# WorldForge Agent 中文协作规则

> 本文件是用户指定的兼容入口。仓库级权威指令仍为 `AGENTS.md`，中文快速入口为 `agent.md`。

## 提交、推送与合并说明语言

1. 人工编写的 Git 提交标题和提交说明必须使用中文。
2. Pull Request 的标题与描述必须使用中文。
3. 推送、同步、合并、关闭、回滚和诊断操作中面向人的标题、摘要、说明与评论必须使用中文。
4. 不得直接使用完整英文句子作为提交标题、PR 标题或描述；`fix`、`feat`、`chore` 等英文类型前缀统一改为“修复”“功能”“维护”“文档”“测试”“重构”等中文表述。
5. 代码标识符、文件路径、分支名、命令、协议名、库名和无法准确翻译的技术专名可以保留英文，但其上下文说明必须使用中文。
6. CI、GitHub 或第三方工具自动生成且无法配置的固定英文状态名不属于人工标题与描述；可配置内容仍必须改为中文。
7. 提交或创建 PR 前必须复查标题和描述语言；发现英文标题或英文说明时，先修正再推送、合并或关闭。
