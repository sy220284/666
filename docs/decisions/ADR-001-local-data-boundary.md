# ADR-001：本地数据边界

- 状态：Frozen
- 日期：2026-07-13

## 背景

WorldForge面向单作者长篇创作。正文、设定、版本和日志具有高隐私性，且用户已明确不建设WorldForge云存储、云同步和模型请求中转。

## 决策

1. 正文、设定、数据库、索引、日志、备份、Prompt、Eval和配置均保存在用户本机。
2. 外部模型请求由Core Service从本机直接发往用户配置的端点。
3. WorldForge不提供账号后台、作品托管、云同步或请求代理。
4. 本地模型由用户自行安装和运行；WorldForge只连接已运行的兼容API。
5. API凭据保存在操作系统凭据服务，项目数据库只保存`credentialRef`。
6. 用户可以自行将项目目录放入其信任的同步盘，但WorldForge不管理或承诺该同步行为。

## 结果

### 正面

- 数据主权清晰。
- 无服务端运营成本和数据泄露面。
- 离线编辑不受影响。
- 本地API与外部API使用统一Provider配置。

### 代价

- 多设备同步、在线协作和远程访问不属于产品能力。
- 用户需要自行管理本机磁盘、系统备份和模型服务。

## 强制约束

- Renderer不得直接调用外部模型或读取凭据。
- 默认日志不得记录正文、完整Prompt和凭据。
- 诊断包必须列出内容并由用户确认。
- 新功能若需要上传项目数据，必须先替代本ADR，不得隐式加入。

## 验证

- 网络断开时，编辑、搜索、版本、备份和导出仍可使用。
- 抓取默认网络流量时，只允许用户主动触发的Provider请求和明确外链。
- 项目目录外不存在正文副本，系统凭据库除外且只含密钥。
