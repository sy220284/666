# ADR-002：SQLite唯一数据真源

- 状态：Frozen
- 日期：2026-07-13

## 背景

WorldForge包含正文、设定、状态、版本、伏笔、搜索索引和AI候选。若编辑器状态、JSON文件、缓存和数据库并列为真源，将出现恢复、迁移和冲突判断不一致。

## 决策

1. 每个项目的`project.sqlite`是该项目唯一权威数据源。
2. `app.sqlite`只保存应用级配置、最近项目和Provider元数据，不保存正文。
3. Tiptap文档、Zustand状态、FTS5索引、摘要、AI日记、导出文件和缓存均为可重建派生数据。
4. 所有项目写入仅由Core Service执行，并经过单一串行写队列。
5. 数据库启用WAL、外键、`busy_timeout=5000`和明确事务边界。
6. FTS5虚表和统计缓存损坏时应重建，不反向覆盖业务表。

## 结果

### 正面

- 备份、恢复和Migration边界明确。
- Renderer重启不会造成数据真源分叉。
- Candidate采用、Version创建和状态确认可使用原子事务。

### 代价

- Core成为唯一写入瓶颈，需要量化性能预算。
- 原生SQLite依赖增加跨平台打包测试成本。

## 强制约束

- Renderer、Main和Preload不得直接打开项目数据库。
- Repository不得绕过写队列执行业务写操作。
- 每个写命令必须验证`projectId`与活动数据库匹配。
- Migration发布后只追加，不修改已发布脚本。

## 验证

- 删除FTS和缓存后可从业务表重建。
- 自动保存、AI流和备份并发时无丢写和静默覆盖。
- `app.sqlite`检查不出现正文、设定或候选内容。
