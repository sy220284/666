# WorldForge 工作流执行顺序

> 状态：Active

## 标准顺序

```text
活动任务与依赖检查
→ 独立任务分支
→ Draft静态检查
→ 同一PR完成任务状态转换
→ Ready全量门禁
→ 等待当前main最终验证成功
→ 串行受控合并
→ 最终main复核
```

## 任务推进前沿

任务索引同时承担执行顺序。自动选择下一张任务时：

1. 找到索引中最后一张状态不为`Planned`的任务，作为执行前沿；
2. 只检查执行前沿之后的第一张`Planned`任务；
3. 依赖满足时激活；
4. 依赖不满足时停止，不跳过到更后的任务；
5. 执行前沿之前残留的旧`Planned`状态不得重新抢占主线。

历史任务返修关闭后，已完成的后续任务仍确定执行前沿，因此系统会恢复到原先暂停的位置，无需维护第二套恢复指针。
