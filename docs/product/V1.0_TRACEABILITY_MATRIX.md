## 维护要求

1. 任务开始时将相关行改为`In Progress`。
2. 代码合并且功能可运行后改为`Implemented`。
3. 对应P0验收与证据完成后才改为`Verified`。
4. 功能延期或删除必须说明原因并同步V1范围、功能清单和路线图。
5. 每个P0需求至少对应一个任务和一个验收ID。
