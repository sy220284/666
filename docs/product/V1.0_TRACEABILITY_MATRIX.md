## 维护要求

1. 任务开始时将相关行改为`In Progress`。
2. 代码合并且功能真实可运行后改为`Implemented`。
3. 对应P0验收、人工复查和证据完成后才改为`Verified`。
4. 功能延期或删除必须同步V1范围、功能清单、路线图和任务卡。
5. 每个P0需求至少对应一张任务卡和一个验收ID。
6. 同一需求跨“基础底座→上层使用→发布回归”时允许映射多张任务卡，但不得重复实现同一权威逻辑。
