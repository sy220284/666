# WorldForge V1.0 范围与验收基线

> 状态：Frozen  
> 设计基线：WorldForge V6.5  
> 更新日期：2026-07-14

## 1. 产品目标

V1.0交付本地桌面长篇写作核心闭环：

```text
创建项目
→ 基础写作与版本
→ 规划、设定与连续性
→ AI候选生成与作者采用
→ 校验、搜索、导入导出
→ 备份、恢复与发布验收
```

无AI时必须能够创建项目、建卷章、中文写作、自动保存、创建Version、导入导出和恢复。

## 2. V1.0 P0范围

| 领域 | 必须实现 |
|---|---|
| 应用与项目 | Electron安全应用；创建、打开、关闭、移动、最近项目；异常项目只读打开 |
| 数据 | `app.sqlite`应用配置；每项目`project.sqlite`权威数据；Migration、完整性检查和单写队列 |
| 基础编辑 | Tiptap块级正文；中文IME；粘贴清理；自动保存；字数；当前章查找；撤销重做 |
| 正文安全 | Draft/Candidate/Version；`logicalBlockId`；Revision；SHA-256 Hash；Block Patch；锁定块 |
| 项目结构 | 卷、章节、SceneBeat；软删除；回收站；拆章、并章和跨章移动 |
| 规划设定 | ProjectBrief；大纲树；人物、地点、势力、道具、能力、规则、事件和自定义实体 |
| 连续性 | Canon与动态状态分离；时间线；知情；伏笔；尾快照；状态提案；人物弧光与里程碑 |
| AI接入 | OpenAI兼容、Anthropic和批准的Custom适配器；外部API或已运行本地服务直连 |
| AI基础 | FTS5；P0—P4约束包；Prompt Registry；Cleaner；GenerationRun；取消和partial Candidate |
| AI写作 | T0骨架、T1扩写、快速改写、结构性改写、融合和候选审阅 |
| 候选采用 | Diff；块级/SceneBeat级采用；冲突处理；原子撤销；采用前恢复点 |
| 校验修订 | 确定性、统计和AI语义校验；StoryTodo与Comment；人物弧光一致性 |
| 搜索节奏 | 全项目FTS5；安全批量替换；项目词典；爽点、钩子、更新节奏和黄金三章建议 |
| 导入导出 | TXT、Markdown、DOCX导入预览；指定Version导出；失败回滚 |
| 数据安全 | 三轨备份；恢复副本；最后备份保护；项目和路径边界 |
| UI与显示 | 新手/专业模式；三条路径；工作台；沉浸写作；帮助；真实进度；双视觉主题；无障碍；目标显示环境 |
| 质量 | 单元、Repository、集成、Migration、安全、E2E、性能、AI Eval和业务场景验收 |

所有节奏指标为P3建议级，可关闭，不阻断生成、保存或定稿。

## 3. V1.0 P1范围

P1可以在P0稳定后独立实施，不阻塞V1.0发布：

- 自定义快捷键。
- 打字机模式。
- 高级三栏冲突视图。
- 研究笔记和本地附件关联。
- 更丰富的批注筛选、标签和批处理。
- Theme B额外护眼/高对比变体。

P1功能不得预先要求P0数据库建立无任务消费的表；启动P1任务时再增加Migration。

## 4. V1.5延期范围

- 完整L0—L5自动记忆调度。
- 热、温、冷数据自动迁移。
- 剧情弧/卷级摘要自动维护。
- 卷级连续性检查点传播。
- 手动与定时AI项目日记。
- 300万—500万字完整适配。
- 经真实评测触发的本地语义检索。
- 可选实时项目数据库加密。
- 作者自定义Theme B印文。

V1.5功能不得在V1.0中提前建立调度器、向量层、日记表或无消费方的数据结构。

## 5. 非目标

- WorldForge云存储、云同步、账号后台或作品托管。
- WorldForge模型请求中转或云端AI服务。
- 本地模型下载、安装、升级和进程监管。
- 无人审核批量生成和自动发布。
- 多人协作、CRDT、插件市场和社区平台。
- 读者反馈运营、平台管理、云部署和云运维体系。
- V1.0项目包`.wfproj`导入导出；V1.0只支持正文和文档格式导入导出。

## 6. 代码硬保证

以下情况必须保持为0：

1. 锁定块被AI或批量操作修改。
2. 未确认Candidate进入活动Draft。
3. Revision或Hash冲突被静默覆盖。
4. AI直接写入Canon、EntityState或ArcMilestone权威状态。
5. 跨项目读写。
6. 凭据进入项目数据库、普通配置或日志。
7. 恢复操作直接覆盖原项目。

## 7. V1.0完成条件

### 功能

- P0功能均实现并通过`P0_ACCEPTANCE_MATRIX.md`中的P0-001—P0-075。
- 每个P0需求均关联任务卡、测试和证据目录。
- 用户可从新建项目完成无AI写作闭环和AI候选闭环。

### 数据

- 新库、旧库升级、升级中断和高版本只读场景通过。
- 自动保存与AI任务并行时无丢写和无静默覆盖。
- Candidate采用、撤销、Version、状态和弧光确认均使用事务。
- 三轨备份可以恢复到新目录。

### 安全

- Renderer无Node、数据库、文件和凭据访问能力。
- Preload只暴露白名单方法，IPC参数全部校验。
- DOCX只提取允许内容，失败后清理临时文件。
- 普通日志不记录正文、完整Prompt和凭据。

### 体验

- 1280×800可完成核心操作。
- 2K 125%缩放下正文清晰且键入P95≤50ms。
- 21:9下正文宽度受控，主要操作靠近工作区。
- AI运行、取消、失败、冲突和恢复状态清楚可见。
- Theme A与Theme B切换不改变业务结果。

### AI

- Provider、模型、任务和Prompt版本分别记录支持等级。
- 未验证模型不标记为完整支持。
- T0/T1不达标时可绕过，不阻断自主写作。

## 8. 范围变更规则

新增或移动功能时必须同步：

1. 本文件。
2. `FUNCTION_CATALOG.md`。
3. `V1.0_TRACEABILITY_MATRIX.md`。
4. `V1.0_ROADMAP.md`和任务卡。
5. `P0_ACCEPTANCE_MATRIX.md`。
6. 对应Schema、IPC、AI、UI和测试专项文档。

只有真实创作任务无法完成且存在可测试验收条件时，才允许扩大P0范围。
