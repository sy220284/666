# WorldForge V1.0 范围与验收基线

> 状态：Frozen  
> 设计基线：WorldForge V6.5

## 1. 产品目标

V1.0交付本地桌面长篇写作核心闭环：

```text
创建项目 → 规划与设定 → 编辑正文 → 生成AI候选 → 审阅采用 → 定稿与连续性维护 → 搜索、备份、恢复和导出
```

成功标准：作者可离线编辑；外部模型由本机直连；AI输出始终可拒绝、可撤销、可追溯；正文和状态不得静默覆盖。

## 2. V1.0 P0范围

| 领域 | 必须实现 |
|---|---|
| 应用与项目 | Electron应用；创建、打开、关闭、移动项目；最近项目；异常项目只读打开 |
| 数据 | `app.sqlite`保存应用配置；每项目`project.sqlite`保存作品权威数据；Migration和完整性检查 |
| 编辑器 | Tiptap块级正文；自动保存；中文输入法安全；粘贴清理；撤销重做；查找；锁定块 |
| 正文模型 | Draft、Candidate、不可变Version；`logicalBlockId`；Revision；块级Patch |
| 项目结构 | 卷、章节、场景；拆章、并章、跨章移动；回收站与恢复 |
| 规划设定 | 最小任务书；大纲树；SceneBeat；人物、地点、势力、道具、能力和规则 |
| 连续性 | Canon与动态状态分离；时间线；知情信息；伏笔；尾快照；状态提案；人物弧光（成长/黑化/觉醒等类型、里程碑状态机、与EntityState一致性校验） |
| AI接入 | OpenAI兼容、Anthropic和明确配置的自定义端点；外部API或已运行本地服务直连 |
| AI生成 | T0骨架、T1扩写、快速改写、结构性改写、融合、取消和部分候选 |
| 候选审阅 | 全屏、单稿和差异视图；块级采用；冲突处理；原子撤销；采用前恢复点 |
| 约束包 | 当前章、场景、尾状态、实体状态、伏笔、知情、作品规则、FTS5补充和Token裁剪 |
| 校验搜索 | 连续性与文风检查；离线基础校对；当前章搜索；全项目FTS5；替换预览；项目词典；网文体裁节奏建议（爽点密度、章末钩子、更新节奏、黄金三章，均为P3建议级不阻塞） |
| 导入导出 | TXT、Markdown、DOCX导入预览与导出；原子提交；失败回滚 |
| 数据安全 | WAL单写队列；三轨备份；恢复副本；数据库检查；项目路径边界 |
| UI与显示 | 新手/专业模式；三种创作路径；沉浸写作；帮助；真实AI进度；1280×800、2K、21:9和混合DPI |
| 质量 | 单元、集成、Migration、安全、E2E、性能、AI Eval和业务场景验收 |

## 3. V1.0 P1范围

- 研究笔记和本地附件关联。
- 段落批注与StoryTodo。
- 自定义快捷键与打字机模式。
- 高对比主题与轻量里程碑。
- 高级三栏冲突视图。
- 手动AI项目日记及章节定稿后的日记候选。

## 4. V1.5延期范围

- 完整L0—L5自动记忆调度。
- 热、温、冷数据自动迁移。
- 剧情弧和卷级摘要自动维护。
- 卷级连续性检查点传播。
- 定时AI项目日记。
- 300万—500万字完整适配。
- 经真实评测触发的本地语义检索。
- 可选实时项目数据库加密。

## 5. 非目标

- WorldForge云存储、云同步、账号后台或作品托管。
- WorldForge模型请求中转或云端AI服务。
- 本地模型运行时的下载、安装、升级和监管。
- 无人审核批量生成和自动发布。
- 多人协作、CRDT、插件市场和社区平台。
- 读者反馈运营、平台管理、云部署和云运维体系。

## 6. 代码硬保证

以下情况必须保持为0：

1. 锁定块被AI或批量操作修改。
2. 未确认Candidate进入活动Draft。
3. Revision冲突被静默覆盖。
4. AI直接写入静态Canon或权威状态。
5. 跨项目读写。
6. 凭据进入项目数据库、普通配置或日志。
7. 恢复操作直接覆盖原项目。

## 7. V1.0完成条件

### 功能

- P0功能均实现并通过验收。
- 每个P0需求均关联任务卡和测试用例。
- 用户可从新建项目完成一章完整创作闭环。

### 数据

- 新库、旧库升级、升级中断和恢复测试通过。
- 自动保存与AI任务并行时无丢写和无静默覆盖。
- Candidate采用、撤销、Version和状态确认均使用事务。
- 三轨备份可以恢复到新目录。

### 安全

- Renderer无Node、数据库、文件和凭据访问能力。
- Preload只暴露白名单方法，IPC参数全部校验。
- DOCX仅提取允许内容，临时文件失败后清理。
- 默认日志不记录正文、完整Prompt和凭据。

### 体验

- 1280×800可完成核心操作。
- 2K 125%缩放下正文清晰且键入P95不高于50ms。
- 21:9下正文宽度受控，主要操作靠近工作区。
- AI运行、取消、失败、冲突和恢复状态清楚可见。

### AI

- 代码硬保证全部通过。
- Provider、模型和任务分别记录支持等级。
- 未验证模型不标记为完整支持。
- T0/T1不达标时可绕过，不阻断自主写作。

## 8. 范围变更规则

新增P0功能时必须同步更新：本文件、功能清单、追踪矩阵、路线图、任务卡和验收矩阵。只有真实创作任务无法完成且存在可测试验收条件时，才允许扩大V1.0范围。
