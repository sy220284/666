# WorldForge V1.0 开发路线图

> 状态：Frozen  
> 原则：先验证风险，再建设核心；先完成M0—M3写作闭环，再补齐M4交付能力，最后进入M5硬化。

## 1. 总体顺序

```text
M0 工程、安全、数据库、IPC和显示原型
→ M0.5 AI质量与中文Diff验证
→ M1 项目、编辑器、Draft/Candidate/Version
→ M2 规划、设定和连续性
→ M3 Provider、约束包、生成和候选审阅
→ M4 校验、搜索、导入导出、备份和完整UI
→ M5 安全、性能、迁移与发布验收
→ V1.0
→ 真实作者使用与长篇数据收集
→ V1.5单独立项
```

## 2. 里程碑总表

| 里程碑 | 目标 | 进入条件 | 退出条件 |
|---|---|---|---|
| M0 | 建立可编译、可测试、安全隔离的工程底座 | 设计、AGENTS和P0文档冻结 | Monorepo可构建；安全窗口、Core、SQLite写队列、流式IPC和高分屏Spike通过 |
| M0.5 | 验证最大技术风险 | M0可运行 | T0/T1最小Eval、中文长段Diff和Provider Stub形成可复测基线 |
| M1 | 建立不可丢稿、不可静默覆盖的编辑与版本核心 | M0退出 | 项目、Draft、Block Patch、锁定、Candidate、Version、撤销和回收站闭环 |
| M2 | 建立规划、设定和连续性真源 | M1退出 | 大纲、场景、实体、Canon、状态、时间线、知情、伏笔、尾快照可用 |
| M3 | 建立作者可控AI写作闭环 | M2退出 | Provider、约束包、T0/T1、快速改写、结构性候选、Diff、冲突和采用闭环 |
| M4 | 补齐日常创作交付功能 | M3退出 | 校验、搜索、词典、导入导出、三轨备份、恢复、主题和完整工作台闭环 |
| M5 | 完成发布硬化 | M4退出 | P0验收、安全、Migration、性能、2K/21:9、跨平台构建和文档一致性通过 |

## 3. M0：工程与安全底座

### M0-01 Monorepo与质量工具

交付：pnpm workspace、Electron三层结构、六个packages、TypeScript strict、Lint、Vitest、Playwright、统一脚本。

### M0-02 Electron安全基线

交付：`contextIsolation`、sandbox、关闭Node集成、严格CSP、preload白名单、导航与外链限制、IPC输入校验。

### M0-03 SQLite与单写队列

交付：`app.sqlite`/`project.sqlite`分离、WAL、`busy_timeout`、Migration Runner、事务封装、写队列、检查点和完整性检查。

### M0-04 IPC与事件协议

交付：命令信封、错误码、MessagePort流式事件、序号、批量、背压、取消和任务隔离。

### M0-05 显示与窗口恢复Spike

交付：1280×800、2K 100/125/150%、3440×1440、3840×1600、混合DPI、DIP坐标和响应式侧栏。

## 4. M0.5：关键风险验证

### M0.5-01 AI最小质量验证

- 固定Provider Stub。
- T0结构化输出Schema。
- T1骨架遵循测试。
- 取消、超时和部分候选。
- 支持等级记录。

### M0.5-02 中文Diff验证

- `logicalBlockId`结构匹配。
- 中文字符级差异。
- 5000字首屏与完整Diff预算。
- 拆分、合并、新增和删除块。

M0.5不建设完整产品界面，只确认技术方案可行。

## 5. M1：编辑与数据核心

### M1-01 项目工作空间

创建、打开、关闭、移动、最近项目、只读异常打开、路径边界。

### M1-02 Draft与编辑器

Tiptap Schema、DraftBlock、Patch、自动保存、IME、粘贴清理、查找和字数统计。

### M1-03 锁定与Revision

UI/Core双层锁定、Hash校验、Revision冲突、批量操作跳过锁定摘要。

### M1-04 Candidate与Version

Candidate隔离、不可变Version、采用事务、整体撤销、采用前检查点和历史恢复。

### M1-05 回收站与结构操作

软删除、恢复、永久删除检查、拆章、并章、场景跨章移动和重大操作恢复点。

## 6. M2：规划与连续性

### M2-01 规划结构

任务书、PlotNode、Volume、Chapter、SceneBeat、拖拽顺序和正文单向安全关联。

### M2-02 设定与状态

Entity、CanonFact、EntityState、来源证据、当前值与历史值分离。

### M2-03 时间线、知情和伏笔

时间事件、依赖关系、人物知情状态、伏笔生命周期和章节关联。

### M2-04 定稿、状态提案与尾快照

定稿Version、状态提案确认、EndingSnapshot、旧章返修后的stale传播。

## 7. M3：AI生成闭环

### M3-01 Provider与凭据

OpenAI兼容、Anthropic、自定义协议；连接测试；系统凭据存储；模型支持档案。

### M3-02 约束包

P0—P4优先级、结构关联、尾快照、FTS5、时序过滤、去重、Token预算和裁剪日志。

### M3-03 T0/T1与改写

骨架、正文、快速改写、结构性改写、融合、流式进度、取消和partial Candidate。

### M3-04 候选比较与采用

全屏比较、单稿、只看差异、折叠未改段、同步滚动、块级采用、冲突和原子撤销。

## 8. M4：完整交付

### M4-01 校验与修订

确定性、统计、AI语义三层校验；问题锚点、忽略、静音、降级、误报和StoryTodo。

### M4-02 搜索与词典

当前章搜索、FTS5、批量替换预览、锁定跳过、项目专名词典。

### M4-03 导入导出

TXT/Markdown/DOCX导入预览；编码处理；DOCX安全降级；Version导出；原子文件写入。

### M4-04 备份恢复

日常滚动、重大操作、手动快照三轨；完整性检查；Hash；恢复到新目录；空间清理。

### M4-05 完整UI

新手/专业模式、三种创作路径、写作工作台、沉浸视图、帮助、状态仲裁、主题和高分屏布局。

## 9. M5：发布硬化

- 全部P0追踪闭环。
- 数据库升级与中断恢复。
- Electron与IPC安全测试。
- 真实业务E2E。
- AI代码硬保证为0。
- 2K、曲面、混合DPI和1280×800验收。
- 性能预算通过。
- Windows、macOS、Linux构建验证。
- README、用户文档和发布检查同步实际实现。

## 10. V1.5启动门

V1.5不能按日期自动开始，必须满足：

1. V1.0已被真实作者稳定使用。
2. 已有至少中型长篇项目数据。
3. FTS5召回不足有可重复评测证据。
4. 自动记忆和定时日记能证明降低维护成本。
5. Core性能仍有余量或已达到拆分阈值。

## 11. 路线图变更规则

- 不允许跳过M0安全和数据底座直接开发业务页面。
- M0.5验证失败时，调整实现方案，不用UI包装掩盖技术风险。
- 单个任务只能跨一个核心领域；跨数据库、IPC和UI的端到端任务必须单独批准。
- V1.5能力不得提前混入V1.0任务。
