# WorldForge 隐私与日志规范

> 状态：Frozen

## 1. 隐私原则

1. 项目内容默认只保存在用户本机。
2. 只有用户主动触发AI任务时，相关约束包才从本机发往用户配置的Provider。
3. WorldForge不提供请求中转、账号后台、云同步和作品托管。
4. 日志、诊断、统计和测试证据遵循最小化原则。

## 2. 数据分类

| 等级 | 数据 | 默认处理 |
|---|---|---|
| S0 公开 | 应用版本、开源许可证 | 可进入日志和文档 |
| S1 运行元数据 | Run ID、模型名、耗时、Token计数、错误码 | 可记录，默认本地 |
| S2 项目元数据 | 项目ID、章节ID、字数、状态 | 仅本地记录，诊断导出前列明 |
| S3 创作内容 | 正文、设定、Prompt、批注、日记、研究资料 | 默认不进入普通日志 |
| S4 凭据 | API密钥、令牌、系统凭据引用的实际值 | 只进入系统凭据库和请求内存 |

## 3. 允许记录的字段

默认结构化日志可以包含：

- 时间、级别、组件和事件名。
- requestId、taskId、runId、projectId的不可逆或内部标识。
- Provider ID、模型名称和协议。
- 延迟、重试次数、Token统计和字符计数。
- 数据库操作类型、行数和耗时。
- 错误码、可重试标记和安全诊断ID。
- 内容Hash，但不保存原文。

## 4. 禁止默认记录

- 正文段落和完整章节。
- 完整System Prompt、约束包和模型原始响应。
- 人物、设定、伏笔和项目日记全文。
- API密钥、认证头、Cookie和系统凭据值。
- 未脱敏本地绝对路径、用户名和设备名称。
- DOCX或附件原始内容。

调试版本也不得用长期日志绕过这些限制。

## 5. 摘录边界

普通日志不保存内容摘录。专门的用户授权诊断允许保存最小局部样本时，必须：

1. 在界面列明将包含的内容。
2. 默认关闭。
3. 允许用户预览和删除条目。
4. 设定字符上限并移除相邻无关正文。
5. 不包含凭据。

“只记录500字”仍属于创作内容，不能当作普通元数据。

## 6. 日志文件

建议：

```text
<project>.worldforge/logs/
├── app-YYYY-MM-DD.jsonl
├── core-YYYY-MM-DD.jsonl
└── diagnostics/
```

项目日志可以只保存项目内ID；应用级日志不得复制正文。默认滚动保留14天或受空间上限约束，用户可手动清理。

## 7. 错误处理

- Renderer只获得标准错误码和安全文案。
- SQL、堆栈、内部路径和Provider原始响应只进入本地受控诊断。
- 认证错误不得回显请求头。
- 数据库错误不得附带正文值。

## 8. 诊断包

诊断包默认包含：

- 应用版本、操作系统和显示器信息。
- 配置Schema，不含凭据值。
- 最近日志中的S0—S2字段。
- Migration版本和数据库检查结果。
- 测试或故障事件ID。

默认不包含：项目数据库、正文、Prompt、备份、附件和模型响应。

导出前必须显示清单，用户明确确认后生成本地文件。WorldForge不自动上传。

## 9. AI请求可见性

AI任务界面应允许作者查看本次将发送的信息类别：

- 当前选区或章节。
- 相关人物与设定。
- 前文尾状态。
- 文风样本。
- 规则与伏笔。

V1不要求展示完整内部Prompt编辑器，但必须让作者知道哪些项目内容会离开本机。

## 10. 测试

- 扫描默认日志，确认不含测试正文、Prompt样本和密钥。
- 故障路径、Provider错误和数据库异常同样执行扫描。
- 诊断包生成前后对文件内容做自动化白名单检查。
- Renderer状态快照不保存credential值。
