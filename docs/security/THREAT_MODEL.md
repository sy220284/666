# WorldForge 威胁模型

> 状态：Approved  
> 适用：V1.0桌面单用户、本地优先架构

## 1. 保护资产

- 未发布作品正文和历史版本。
- 人物、世界观、时间线、伏笔和研究资料。
- 项目数据库、备份与导出文件。
- API凭据。
- 作者手动锁定内容。
- 项目路径与本地附件。
- AI请求中的上下文与Prompt。

## 2. 信任边界

```text
用户
  ↓
Renderer（不可信输入层）
  ↓ 白名单Preload
Electron Main / Core Service（受信任执行层）
  ↓
SQLite / 项目文件 / OS凭据库
  ↓ 用户主动配置
外部API或本地模型服务（外部信任域）
```

Renderer展示的数据和所有用户输入均需验证。Provider服务无论在本机还是外部，都不视为权威数据源。

## 3. 主要威胁与缓解

| ID     | 威胁                       | 影响                | 缓解                                                 |
| ------ | -------------------------- | ------------------- | ---------------------------------------------------- |
| TM-001 | Renderer获得Node或文件能力 | 任意读取本机文件    | 关闭Node集成、启用隔离与sandbox、最小Preload         |
| TM-002 | 任意IPC通道或输入未校验    | 越权调用Core能力    | 具名白名单、Zod校验、协议版本和命令注册表            |
| TM-003 | 路径穿越或符号链接越界     | 读取/覆盖项目外文件 | Core规范化路径、真实路径检查、允许根目录校验         |
| TM-004 | 跨项目ID混用               | 作品数据串写        | activeProjectId、实体归属校验、Repository强制作用域  |
| TM-005 | AI直接覆盖Draft            | 作者正文丢失        | Candidate隔离、明确接受、Revision/Hash/LockGuard     |
| TM-006 | 旧Candidate覆盖新编辑      | 静默回退作者修改    | baseRevision、expectedHash、冲突界面                 |
| TM-007 | 锁定块被批量操作修改       | 关键文本破坏        | Tiptap事务过滤、Core统一LockGuard、整批Patch回滚      |
| TM-008 | DOCX解包异常内容           | 资源耗尽或越界写入  | 限制大小、数量、压缩比、路径；忽略嵌入对象和外部资源 |
| TM-009 | 凭据写入配置、日志或错误   | 密钥泄露            | OS凭据库、credentialRef、日志脱敏、错误字段白名单    |
| TM-010 | 外部模型端点记录内容       | 作品泄露到第三方    | 明确端点类型、用户主动配置、本机直连、隐私提示       |
| TM-011 | 数据库损坏或断电           | 丢稿                | WAL、单写队列、事务、完整性检查和三轨备份            |
| TM-012 | 恢复覆盖原项目             | 二次数据损失        | 默认恢复到新目录，验证后由作者决定                   |
| TM-013 | 日志或诊断包包含正文       | 隐私泄露            | 默认只记ID、耗时、计数和Hash；导出前列出内容         |
| TM-014 | 远程页面在应用内加载       | 页面利用桌面权限    | 禁止导航、新窗口和远程内容；外链系统浏览器打开       |
| TM-015 | 恶意导出路径               | 覆盖关键文件        | 系统选择器、目标确认、临时文件和原子重命名           |

## 4. Electron安全要求

```ts
webPreferences: {
  nodeIntegration: false,
  contextIsolation: true,
  sandbox: true,
  webSecurity: true,
  preload: PRELOAD_PATH
}
```

还必须：

- 严格CSP，不允许`unsafe-eval`进入正式构建。
- 拦截`will-navigate`、`setWindowOpenHandler`和下载行为。
- 不加载用户提供的HTML为应用页面。
- 正式构建受控开放DevTools。
- Electron Fuses在M0安全Spike中评估并测试。

## 5. 数据库与文件威胁

- 每个项目使用独立数据库和目录。
- 写队列不接受原始SQL、表名或路径作为Renderer输入。
- 备份使用SQLite Online Backup，不直接复制打开中的数据库。
- Migration前创建恢复点，失败后停止写入。
- FTS、摘要和缓存损坏时重建，不反向污染业务表。

## 6. AI边界

- Prompt中的“不要修改锁定段落”只是质量提示，真正保护由Core完成。
- Renderer锁定样式和禁用态不是权限边界；绕过Editor直接调用Core仍执行Revision、Hash、归属与LockGuard校验。
- 同一Patch只要存在一个锁定冲突就不持久化任何操作；IPC仅返回冲突类型、logicalBlockId和跳过计数，不返回正文。
- 模型输出必须经过Schema解析和业务校验。
- AI状态提取只生成提案。
- AI日记、摘要和校验均为派生数据。
- 未验证模型不允许绕过代码硬保证。

## 7. 剩余风险

V1.0不解决：

- 设备被完全控制后的本地数据保密。
- 用户自行配置的不可信模型服务如何处理请求。
- 用户自行将项目放入第三方同步盘后的冲突。
- 未启用磁盘加密时共享设备上的明文数据库访问。

这些风险需要在用户文档中明确，不得误导为“绝对安全”。

## 8. 安全验收

- Renderer访问`require`、`process`、数据库和文件系统失败。
- 未注册IPC和非法字段被拒绝。
- 路径越界、符号链接逃逸和跨项目ID测试失败关闭。
- 锁定、Candidate隔离和Revision冲突硬保证为0缺陷。
- 导入异常文件不写项目、不留下临时内容。
- 默认日志和诊断包扫描不含正文与凭据。

## M1-09 本地文本文件边界

- Renderer无任意路径能力；源文件和目标目录只由Electron Main系统选择器产生。
- Core拒绝非绝对路径、符号链接、非普通文件、未知扩展、二进制NUL、空内容和20 MiB以上输入。
- 自动检测采用严格解码；低置信度GB18030必须在预览界面可见并允许人工重选。
- ImportPlan绑定源文件SHA-256并设置30分钟有效期，文件变化或过期阻止提交。
- 导出文件名禁止路径分隔符、控制字符和`..`；同名文件绝不覆盖。
- 导出经同目录临时文件、内容Hash验证和原子重命名完成，失败清理临时文件。
