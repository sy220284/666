# 1. 执行摘要与产品边界

WorldForge是一款面向单个作者的本地AI长篇写作工作站。它不承担模型安装、云端托管、读者运营和团队治理，而专注于长篇写作最难的几件事：结构规划、AI候选生成、作者可控编辑、跨章连续性、版本回滚和最终导出。

## 1.1 五项不可变原则

1.  作品正文、设定、索引、日志和备份仅保存在用户本机。

2.  AI结果首先是Candidate，未经作者接受不得覆盖当前Draft。

3.  project.sqlite是单项目唯一权威数据源，缓存和索引均可重建。

4.  锁定块、Revision、不可变Version和项目边界由代码校验，而不是依赖Prompt。

5.  长期记忆中的摘要、AI日记和检索结果均为派生信息；不得覆盖定稿正文、作者确认设定和当前权威状态。

6.  AI可以提出状态变化和问题，但静态设定、定稿正文与权威状态仍由作者确认。

## 1.2 v1完成标准

- 作者可以创建作品、人物、世界规则、大纲、章节和场景节拍。

- 作者可以调用外部API或已运行的本地兼容API生成骨架、正文和局部重写候选。

- 候选稿可以比较、部分采用、融合和丢弃，且不会静默覆盖作者正文。

- 章节定稿后可以提取动态状态变化，作者确认后形成下一章连续性快照。

- 历史章节返修会提示后续快照失效，但不会自动改写后续正文。

- 项目可以离线编辑、全文检索、备份、恢复和导出。

- 小白作者可以通过向导在五分钟内建立最小项目并进入第一章，不需要理解工程术语。

- 资深作者可以跳过任务书、大纲和AI生成，直接进入正文编辑并按需调用局部工具。

- 作者可以在同一写作工作台完成编辑、上下文查看、AI操作、候选比较和状态确认。

- 超长篇项目可以按章节、剧情弧、卷、当前状态和历史账本分层装载记忆，旧状态不会被误当成当前事实。

- 作者可以手动或按本地计划生成AI项目日记；日记记录变化并链接来源，但不自动修改权威设定。

## 1.3 明确非目标

- 不建设云存储、云同步、账号后台、作品托管或WorldForge请求中转。

- 不下载、安装、升级或监管本地大模型运行时；用户自行使用Ollama、LM Studio、llama.cpp server或其他服务。

- v1不做向量数据库、Embedding、Rerank、MCP、CRDT、插件市场和多人实时协作。

- 不做读者反馈聚类、平台运营分析、自动发布、无人审核批量生成和AI检测规避。

- 不自动学习用户创作偏好；作者通过可编辑的作品规则明确约束。

# 2. 用户与核心创作闭环

## 2.1 目标用户

目标用户是独立完成长篇创作的个人作者，覆盖第一次系统写长篇的小白作者，以及已有成熟工作流的资深作者。系统不以作者水平划分数据模型，而通过界面渐进披露降低复杂度：小白获得向导、模板和下一步提示；资深作者获得自由写作、完整上下文和精确控制。

## 2.2 双模式与三种创作路径

新手模式与专业模式只改变默认入口、字段显隐和操作提示，不创建两套数据，也不限制任何核心能力。作者可以随时切换，切换后正文、设定、版本和连续性信息保持一致。

| **维度** | **新手模式**                         | **专业模式**                              |
|----------|--------------------------------------|-------------------------------------------|
| 首页     | 显示继续写作、建书步骤和下一步建议   | 显示最近章节、项目概览和快捷入口          |
| 规划     | 问题式向导、少量必填项、结构模板     | 完整任务书、大纲树、场景卡和高级字段      |
| 写作     | 常用AI按钮、解释性提示、隐藏技术术语 | 完整上下文、来源审计、高级约束和批量操作  |
| 连续性   | 默认折叠，仅显示高风险提示           | 时间线、知情、伏笔和状态面板完整开放      |
| 术语     | 当前稿、AI备选稿、历史版本           | 可显示Draft、Candidate、Version等审计术语 |

系统支持三种并列创作路径，任何章节都可单独选择，不要求全书保持同一模式：

| **路径** | **适用作者**                       | **最短流程**                      |
|----------|------------------------------------|-----------------------------------|
| 自主写作 | 作者自己完成正文，AI只在需要时介入 | 直接写作 → 检查 → 定稿            |
| 混合创作 | 作者掌握结构，AI完成局部场景或过渡 | 节拍 → 局部生成 → 作者修改 → 定稿 |
| AI初稿   | 作者先选结构方案，再让AI扩写整章   | T0骨架 → T1正文 → 候选比较 → 定稿 |

# 4. 精简总体架构与技术选型

> **图示说明：** Image: image2.png


## 4.1 运行进程

| **进程**                     | **职责**                                               | **禁止事项**                                   |
|------------------------------|--------------------------------------------------------|------------------------------------------------|
| Electron Main                | 窗口、生命周期、系统菜单、凭据入口、Core进程监管       | 不执行重型SQL、不保存正文、不直接调用模型      |
| Renderer                     | React界面、编辑器、状态展示、用户操作                  | 不直接访问SQLite、文件系统和密钥               |
| Core Service Utility Process | SQLite唯一写入、文件、FTS5、AI调用、生成流、校验、备份 | 不把正文上传到WorldForge服务，不修改非目标项目 |

## 4.2 最终技术栈

| **层级** | **技术**                              | **理由**                                 |
|----------|---------------------------------------|------------------------------------------|
| 桌面端   | Electron                              | Chromium一致性、Node能力、适合复杂编辑器 |
| 前端     | React + TypeScript + Vite             | 生态成熟、类型共享、AI编程友好           |
| 状态管理 | Zustand                               | 轻量、局部状态清晰                       |
| 编辑器   | Tiptap + ProseMirror                  | 结构化文档、事务、节点属性和扩展能力     |
| 数据库   | SQLite + better-sqlite3               | 本地单文件、事务可靠、FTS5、在线备份     |
| 数据校验 | Zod + JSON Schema                     | IPC、AI结构化输出和配置统一验证          |
| 进程通信 | Electron IPC / MessagePort            | 无需本地HTTP服务和端口管理               |
| 测试     | Vitest + Playwright                   | 单元、集成和桌面E2E覆盖                  |
| 构建     | pnpm workspace + electron-builder     | 单仓库管理和桌面打包                     |
| 文本差异 | logicalBlockId结构Diff + 块内文本Diff | 支持中文长段落的稳定逐块采用             |
| 基础校对 | 本地规则引擎 + 项目词典               | 离线可用，不依赖模型且不自动改正文       |

## 4.4 不预建的基础设施

- 不内置Local Runtime Manager。设置页只提供API地址、模型名、密钥和连接测试。

- 不定义VectorSearchAdapter。v1只实现结构化关联和FTS5。

- 不拆分独立AI Orchestrator进程。出现明确性能或隔离问题后再拆。

- 不引入完整OpenTelemetry。只保存必要的本地运行日志和调用统计。
