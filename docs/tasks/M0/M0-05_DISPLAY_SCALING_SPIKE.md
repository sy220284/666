# M0-05 2K、曲面屏与窗口恢复Spike

> 状态：Planned  
> 优先级：P0  
> 分支：`feat/m0-display-scaling-spike`

## 目标

在正式页面开发前验证1280×800、2K、21:9和混合DPI下的布局、缩放与窗口恢复策略。

## 依赖

M0-01。

## 关联

- 需求：REQ-041
- 验收：P0-063—P0-066

## 必读文档

- `docs/ui/RESPONSIVE_AND_DPI.md`
- `docs/ui/UI_SYSTEM.md`
- `docs/testing/PERFORMANCE_BUDGETS.md`

## 实施内容

1. 保存DIP窗口坐标、displayId、scaleFactor和最大化状态。
2. 实现680/760/860 CSS px正文版心原型。
3. 界面缩放、正文字号和正文宽度独立。
4. `<1100px`右抽屉，`<900px`左右双抽屉。
5. 21:9工作区居中、偏左、偏右原型。
6. 候选比较双栏、上下和单稿占位原型。
7. 验证Popover、选区工具和对话框跨屏重定位。

## 测试环境

- 1280×800。
- 2560×1440：100%、125%、150%。
- 3440×1440。
- 3840×1600。
- 混合DPI双屏。
- 1024×640有效视口。

## 完成条件

无整页水平滚动；文字与SVG清晰；工作区操作距离合理；窗口跨屏与重启后可见且布局正确。
