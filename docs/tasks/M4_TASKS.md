# M4 任务卡：完整交付能力

## M4-01 校验、问题降噪与修订待办

- 目标：提供可解释、可忽略、不会喧宾夺主的质量检查。
- 依赖：M3。
- 分支：`feat/m4-validation-revision`
- 关联：REQ-031，P0-043、044。

### 实现

- 确定性校验：必选节拍、引用、时间顺序、锁定与格式。
- 统计校验：句长、段长、对话比例、重复符号、目标字数。
- AI语义校验：人物行为、设定偏离、衔接和文风风险。
- ValidationIssue锚点、依据、建议和状态。
- 本章忽略、项目静音、降低等级和误报记录。
- 一键转StoryTodo。
- 状态仲裁：安全问题优先，首页主动提示最多2条。

### 测试

相同输入确定性结果一致；AI问题不自动改正文；忽略和静音范围正确；旧Version问题标记stale。

## M4-02 当前章搜索、FTS5、替换与词典

- 目标：完成从单章到全项目的安全检索和批量修订。

## M4-03 TXT、Markdown与DOCX导入导出

- 目标：安全迁移已有作品并稳定交付定稿文本。
- 依赖：M1、M2。
- 分支：`feat/m4-import-export`
- 关联：REQ-034、035，P0-048—050。

### 实现

- TXT编码候选：UTF-8、UTF-16、GB18030；低置信度人工选择。
- Markdown基础结构解析。
- DOCX只提取段落、标题和允许的基础格式。
- 解包总大小、文件数、压缩比和路径限制；忽略嵌入对象和外部资源。
- ImportPlan：分章预览、合并、拆分、重命名和取消。
- 提交前恢复点，单事务导入。
- 从选定Version导出TXT、Markdown和DOCX。
- 临时文件+原子重命名。

### 测试

不同编码、空文档、异常DOCX、取消、分章调整、目标文件冲突、导出失败和内容往返一致性。

## M4-04 三轨备份、完整性检查与恢复

- 目标：覆盖日常误操作、高风险变更和数据库损坏。
- 依赖：M1、M0-03。
- 分支：`feat/m4-backup-recovery`
- 关联：REQ-036、037，P0-051—055。

### 实现

- SQLite Online Backup。
- 日常滚动备份，默认14份。
- 迁移、导入、替换、拆并章前的重大恢复点，默认永久。
- 手动命名快照。
- 备份完成后`integrity_check`和Hash。
- 最后一份已验证备份保护。
- 恢复到新目录并注册为新项目。
- 空间统计和安全清理。

### 测试

写入期间备份、空间不足、备份损坏、删除保护、恢复目标冲突、恢复中断和恢复后完整创作流程。

## M4-05 新手/专业模式、工作台与完整视觉交互

- 目标：将已接通能力形成清晰、安静、长期可用的桌面产品。
- 依赖：M1—M4-04。
- 分支：`feat/m4-complete-ui`
- 关联：REQ-038—041，P0-057—066。

### 实现

- 新手和专业模式共用数据，只改变披露度。
- 自主写作、混合创作和AI初稿三条路径。
- 对话式新建向导和跳过步骤后的轻量脚手架。
- 写作工作台三栏与侧栏折叠。
- 沉浸写作视图。
- 候选全屏比较、冲突页和快速改写内联交互。
- 上下文帮助、首次提示和统一用户术语。
- 浅色、深色、护眼、高对比主题。
- 2K、21:9、混合DPI和窗口恢复。
- 键盘、焦点顺序、中文输入和减少动态效果。

### 视觉原则

安静编辑部；正文为中心；AI来源使用低饱和蓝紫，不用绿色暗示更优；普通设置减少卡片化；微动效120—200ms。

### 验收

按P0-057—066逐项截图和操作；13寸、2K 125%、21:9和混合DPI均完成核心业务流程。
