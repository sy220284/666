# M5-00 作者工作流与产品体验收口

> 状态：Planned  
> 里程碑：M5 AI生成与候选审阅  
> 优先级：P0  
> 建议分支：`feat/m5-author-workflow-experience`  
> 来源：产品使用逻辑与用户体验审计、Issue #188

## 目标

在接入T0/T1及后续AI生成界面前，以真实网文作者的日常工作流为主线，完成现有基础写作、规划、设定、恢复与结构操作的产品体验收口，确保复杂底层能力以作者可理解、可发现、可恢复的方式呈现。

## 阶段定位

本任务是M5的前置体验门。M4完成AI基础设施后，必须先完成本任务，M5-01及后续AI生成与候选审阅任务才能进入实施。

本任务不推翻M0—M4建立的Core合同、权威数据模型、Revision/Hash、Candidate隔离、恢复点和项目边界；只在必要时补充用户状态持久化、查询合同和Renderer交互能力。

## 非目标

- 不实现T0、T1、改写、融合或模型调用。
- 不实现M6全项目搜索、校验、DOCX和三轨备份完整功能。
- 不完成M7双主题视觉精修、全量无障碍和所有显示环境终验。
- 不以隐藏技术信息为由削弱诊断、错误码、数据安全或可观测性。
- 不建立第二套新手数据模型、临时旁路存储或Renderer权威数据源。

## 实施内容

### 1. 建立作者语言层

1. 建立集中式用户术语映射，普通界面使用“当前稿、历史版本、备选稿、已确认设定、待确认变化、写作服务”等作者语言。
2. `Core`、`Revision`、`Hash`、`DraftBlock`、`project.sqlite`、`app.sqlite`、`Renderer`、`Zustand`等工程术语只在高级诊断、冲突详情或明确技术帮助中展示。
3. 统一按钮、空状态、保存状态、取消、失败、只读、冲突和恢复文案；用户取消不得显示为危险错误。
4. 错误界面先说明影响、可执行动作和数据是否安全，再提供错误码与诊断ID。

### 2. 收口主导航与项目操作

1. 一级导航保持首页、写作、规划、设定、检查和设置的任务导向结构，未实现入口不得伪装可用。
2. 顶部仅保留项目身份、保存/任务摘要和高频入口；移动、关闭、恢复、导入导出等低频操作收入项目菜单或对应工作台。
3. 修正恢复、安全、检查路由归属，确保当前导航状态、返回路径和页面标题一致。
4. 所有跨工作台跳转在保存成功后执行，并可返回原章节、光标和滚动位置。

### 3. 实现首页继续写作闭环

1. 持久化最近有效的项目、章节、编辑时间、光标和滚动位置；不得以Renderer内存状态冒充跨重启恢复。
2. 首页提供明确的“继续写作”主行动，一次点击恢复上次项目和章节；路径失效、项目只读或数据异常时进入对应安全分支。
3. 新用户、无项目、项目失效、只读项目和存在后台任务时分别提供可执行空状态。
4. 健康提示必须携带“重新定位、打开恢复中心、重启写作服务、查看详情”等真实处理动作。

### 4. 突出正文与写作效率

1. 写作页保持左卷章、中正文、右上下文结构，正文为默认视觉中心。
2. 左右侧栏可折叠；提供沉浸写作视图状态，不改变业务数据。
3. 查找替换、块类型和低频结构工具按使用频率分级，避免全部操作长期占据正文上方。
4. 自动保存状态持续可见但不抢占注意力；手动保存、复制未保存文本和失败重试路径清晰。
5. 禁用、正在执行、只读、未选择对象和功能不可用使用不同状态及原因，不得统一显示等待光标。

### 5. 设定工作台作者化

1. 人物、地点、势力、道具、能力、规则、事件和自定义类型使用中文名称与说明。
2. 常用设定通过结构化表单编辑；普通模式不得要求作者填写JSON或内部事实键。
3. 保留自定义字段和高级原始数据视图，但必须与普通编辑明确分层。
4. 已确认设定、动态状态、历史记录和待确认提案在视觉与交互上清晰分离。
5. 不改变CanonFact、EntityState和StateProposal的权威边界。

### 6. 高风险结构操作可视化

1. 将拆章、并章和跨章移动从`prompt/confirm`序号输入改为专门交互视图。
2. 拆章支持正文分割位置选择和拆分后双章预览。
3. 并章显示来源章、目标章、顺序、锁定块、关联SceneBeat和恢复点影响。
4. 跨章移动允许选择实际正文块与目标插入位置，并展示前后文和引用影响。
5. 提交继续使用既有预览、`planHash`、LockGuard、原子事务和恢复点，界面不得绕过Core校验。

### 7. 统一新手与专业模式状态

1. 默认模式、页面临时切换和导航披露使用单一权威状态源。
2. 新手与专业模式共用数据、命令和业务组件，只改变信息密度、默认展开项和帮助程度。
3. 消除外层模式与工作台内部模式互相冲突或显示不一致的情况。
4. 模式切换后当前任务、选中对象和未保存输入不得丢失。

### 8. 测试、性能与体验复核

1. 为继续写作、模式切换、导航返回、侧栏折叠、沉浸写作、健康提示动作和高风险预览补齐单元、集成及Electron E2E。
2. 覆盖中文输入、自动保存失败、Core异常、只读项目、路径失效、锁定块和恢复点失败组合场景。
3. 在1280×800完成新建/打开、继续写作、规划、设定、拆并章和恢复路径，不产生整页横向滚动。
4. 不降低现有覆盖率门禁，不引入新的全局状态泄漏、重复请求、竞态或跨项目数据暴露。
5. 完成后按真实作者首次使用与日常续写两条路径重新执行产品体验审计。

## 产品验收场景

### 场景A：首次使用

```text
启动应用
→ 新建项目
→ 选择引导式或空白起步
→ 创建/打开第一章
→ 输入中文正文
→ 看见可信保存状态
→ 关闭并重新打开
→ 一键继续到原位置
```

新用户完成该场景时，无需理解数据库、Core、Revision、Hash、DraftBlock或Candidate内部模型。

### 场景B：日常续写

```text
启动应用
→ 点击继续写作
→ 恢复上次章节、光标和滚动位置
→ 正文输入与自动保存
→ 按需展开设定或上下文
→ 返回正文原位置
```

从启动应用到正文可输入不超过3次明确操作。

### 场景C：高风险调整

```text
选择拆章/并章/跨章移动
→ 可视化选择范围与目标
→ 查看正文、锁定、关联和恢复影响
→ 确认
→ Core原子执行并创建恢复点
→ 可从恢复中心定位结果
```

## 完成条件

- 普通作者界面不再要求理解工程实现术语；技术信息仍可从高级诊断获取。
- 首页可以真实恢复最近项目、章节、光标和滚动位置，不依赖当前进程内存。
- 作者打开应用后3步内可继续正文输入。
- 写作页正文为视觉中心，左右侧栏可折叠，沉浸视图可用。
- 常用设定不再要求输入英文枚举、JSON或内部事实键。
- 拆章、并章和跨章移动均有可视化影响预览，且继续受既有Core安全机制保护。
- 新手与专业模式只有一个权威状态源。
- Quality、Security、Performance、覆盖率与Electron E2E全部通过。
- 产品体验复审未发现阻断M5-01的P0问题。
