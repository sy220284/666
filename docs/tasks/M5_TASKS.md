# M5 任务卡：发布硬化与验收关闭

## M5-01 安全、Migration、数据损坏与隐私硬化

- 目标：将M0—M4的安全和数据设计验证为发布门槛。
- 依赖：M4。
- 分支：`test/m5-security-data-hardening`
- 关联：REQ-001、003—006、024、042、043。

### 工作内容

- 全量Electron安全配置复核。
- Preload白名单和IPC Schema覆盖率检查。
- 项目路径、符号链接和跨项目故障注入。
- 所有Migration Fixture逐级升级和中断恢复。
- `quick_check`、`integrity_check`和备份恢复演练。
- 日志、错误和诊断包敏感内容扫描。
- DOCX异常Fixture与临时目录清理。
- Candidate、锁定、Revision和不可变Version硬保证回归。

### 阻断条件

任一代码硬保证不为0；恢复覆盖原项目；凭据或正文进入默认日志；高版本数据库被写入。

### 证据

安全测试报告、Migration矩阵、损坏恢复记录、日志扫描报告和阻断问题清单。

## M5-02 性能、高分屏、AI Eval与长场景验收

- 目标：确认真实规模和目标显示环境下产品可用。
- 依赖：M4。
- 分支：`test/m5-performance-eval`
- 关联：REQ-026、029、030、041。

### 性能测试

- 2K键入P95≤50ms。
- 自动保存P95≤150ms。
- 编辑IPC P95≤200ms。
- AI取消反馈≤500ms。
- 5000字Diff首屏≤500ms、完整≤1.2s。
- 正文滚动≥50fps。
- Core单次事件循环阻塞<100ms。
- FTS5在中型Fixture中的查询和重建时间。

### 显示测试

1280×800；2560×1440 100/125/150%；3440×1440；3840×1600；混合DPI；1024×640有效视口。

### AI Eval

- Provider、模型、任务和PromptVersion分别记录。
- T0、T1、快速改写、状态提取、连续性和禁止信息泄露。
- 输出已验证、有限支持和未验证档案。
- 未达标任务必须降级，不得通过修改评分掩盖。

### 拆分决策

达到任一量化阈值时，单独评审是否拆分AI Utility Process；不在本任务中顺手重构。

## M5-03 跨平台构建、P0验收与发布关闭

- 目标：完成V1.0发布前的最终核对。
- 依赖：M5-01、M5-02。
- 分支：`docs/m5-release-acceptance`
- 关联：全部V1.0需求和P0-001—070。

### 构建验证

- Windows安装、启动、卸载和原生模块匹配。
- macOS构建、签名/公证流程记录和权限提示。
- Linux目标包启动和系统凭据不可用时的会话级降级。
- 更新安装不得破坏已有项目。

### 业务验收

执行完整场景：

1. 新手创建作品并完成第一章。
2. 资深作者跳过向导直接写作。
3. AI T0/T1、切章、冲突、采用、撤销和定稿。
4. 状态提案、尾快照和下一章约束。
5. 搜索、批量替换、导入、导出、备份和恢复。
6. 2K、21:9、混合DPI和深浅主题。

### 文档关闭

- 功能清单与实际实现一致。
- 追踪矩阵所有P0项达到Verified或有明确阻断说明。
- README只描述真实可用功能。
- 用户快速开始、已知限制、发布检查和变更记录完成。
- 未完成的V1.5内容明确标记延期。

### 发布结论

输出“允许发布 / 有条件允许 / 禁止发布”，并列出阻断问题、风险、缺失证据和已知限制。不得只写“测试通过”。
