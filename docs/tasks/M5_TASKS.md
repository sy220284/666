# WorldForge M5 AI生成与候选审阅任务摘要

> 状态：Frozen  
> 用途：里程碑导航与阶段门说明；不可替代独立任务卡。

## 阶段目标

先完成作者工作流与产品体验收口，再完成T0/T1、改写、融合、候选审阅、采用和撤销的作者可控AI闭环。

## 任务顺序

| ID | 任务 | 依赖 | 核心交付 |
|---|---|---|---|
| M5-00 | [作者工作流与产品体验收口](M5/M5-00_AUTHOR_WORKFLOW_PRODUCT_EXPERIENCE.md) | M4 | 在AI生成界面接入前，统一作者语言、继续写作、工作台层级、设定编辑和高风险结构操作体验。 |
| M5-01 | [T0多候选骨架](M5/M5-01_T0_SKELETON.md) | M5-00 | 基于章节目标、SceneBeat和约束包生成多个可比较的结构化骨架Candidate。 |
| M5-02 | [T1章节扩写](M5/M5-02_T1_CHAPTER_GENERATION.md) | M5-01 | 基于作者选定或编辑后的骨架和完整约束包生成章节Candidate。 |
| M5-03 | [快速改写与结构性改写](M5/M5-03_REWRITE_WORKFLOWS.md) | M5-02、M2-03 | 实现高频单段快速改写和跨段/跨场景结构性改写，保持可预览、可冲突处理和可撤销。 |
| M5-04 | [多候选融合与部分结果恢复](M5/M5-04_CANDIDATE_MERGE_PARTIAL.md) | M5-02、M5-03 | 按SceneBeat组合多个候选并安全处理取消、断流和partial Candidate。 |
| M5-05 | [候选审阅、采用与冲突工作台](M5/M5-05_CANDIDATE_REVIEW_APPLY.md) | M5-01、M5-02、M5-03、M5-04、M2-03 | 将AI候选和已有Diff/采用引擎整合为完整审阅体验。 |

## 阶段退出门

- M5-00产品体验门已Verified，普通作者无需理解工程术语即可完成首次写作与日常续写。
- T0/T1、改写、融合、候选审阅和采用闭环可用。
- 所有AI输出先成为Candidate，失败和取消不改变Draft。
- 长章节Diff和审阅性能达到预算或有明确降级。

## 执行规则

- 只能通过`ACTIVE_TASK.md`激活其中一张任务卡。
- M5-00未Verified前不得激活M5-01或提前叠加AI生成入口。
- 未满足依赖不得提前实现后续任务。
- 每张任务完成后同步追踪矩阵与证据目录。
