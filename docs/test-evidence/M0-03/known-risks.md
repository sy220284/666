# M0-03 已知限制与剩余风险

- `node:sqlite`在当前Node与Electron运行时仍输出ExperimentalWarning。仓库锁定并实测了Host Node 24.14.0/SQLite 3.51.2和Electron 43.1.1内嵌Node 24.18.0/SQLite 3.53.1；跨平台安装包回归继续由M8执行，不能把当前探测外推为所有未来运行时保证。
- `requestId`幂等结果是单进程基础，缓存上限为1000条，重启后不会恢复。M0-04建立命令/任务协议时需定义跨恢复边界的幂等语义；涉及业务持久化的命令仍应使用数据库唯一约束或持久化操作记录。
- 已有Schema升级必须提供并成功执行`prepareRecoveryPoint`，但实际在线备份、校验、恢复到新副本和恢复中心属于M1-08。当前底座会在缺少该能力时安全拒绝升级。
- 单写队列保证同一Core进程内只有一个写通道；架构仍需维持Core为数据库唯一写入所有者，禁止其他进程绕过该入口直接打开写连接。
- checksum、高版本Schema、损坏与迁移失败可安全只读打开，但面向用户的只读提示、项目选择和恢复操作由M1-02/M1-08接通。
- 初始Migration刻意只含底座表，不代表`DATABASE_SCHEMA.md`中的全部V1表已经实现。
