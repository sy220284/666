# M3-02已知风险

1. 兼容合同仍公开`characterIds`与`locationIds`。两组值由数据库触发器与`scene_beat_entities`保持一致；后续新模块应直接读取权威关系表，避免扩大兼容字段用途。
2. 旧项目若含实体缺失、跨项目或类型错误的历史UUID，Schema 15会原子中止升级并进入既有Migration恢复流程，不会静默删除或猜测修复。
3. 已存在的归档实体引用用于历史追溯与永久删除阻断。Renderer标注“已归档”，作者可显式移除，系统不会自动解除。
4. 关系同步依赖集合差分和幂等触发器。后续修改关系表或兼容列时，必须同时运行Migration、Integration与Electron E2E。
5. 原始桌面PNG保存在GitHub Actions工件中，仓库保存不可变工件摘要与原图SHA-256描述。工件受平台保留期约束，长期审计以日志、哈希和可重复E2E为准。
