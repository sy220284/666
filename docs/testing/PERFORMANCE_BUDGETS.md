# WorldForge V1.0 性能预算

> 状态：Approved  
> 目标：在单机单用户场景下保证写作不中断、AI任务不拖慢编辑、2K与曲面屏布局稳定。

## 1. 测量原则

1. 使用P95而非单次最佳值。
2. 报告机器、操作系统、内存、CPU、存储、分辨率、缩放和数据规模。
3. 开发模式与正式构建分别记录，发布门以正式构建为准。
4. AI模型网络耗时不作为应用性能指标，但取消响应、流式渲染和本地处理必须计入。
5. 性能失败不得通过减少数据完整性检查或绕过安全约束修复。

## 2. 交互预算

| 指标 | 目标 | 阻断阈值 |
|---|---:|---:|
| 2K编辑器键入延迟P95 | ≤50ms | >100ms |
| 普通按钮反馈 | ≤100ms | >250ms |
| 标签/抽屉切换 | ≤160ms感知完成 | >300ms |
| 当前章查找首次结果 | ≤100ms | >300ms |
| 章节切换并显示正文 | ≤300ms | >800ms |
| 自动保存事务P95 | ≤150ms | >500ms |
| 编辑IPC往返P95 | ≤200ms | >500ms |
| AI取消确认 | ≤500ms | >1500ms |

## 3. 编辑器预算

测试规模：

- 常规章节：3000—8000字。
- 长章节：20000字。
- 极端章节Fixture：50000字，仅保证可编辑，不要求所有高亮实时开启。

目标：

- 常规章节滚动≥50fps。
- 选择、复制、撤销和输入无明显冻结。
- 自动保存不阻塞输入线程。
- 来源和锁定标记不引发整章重渲染。
- 大章节可降级关闭非必要实时统计，但不降低正文安全。

## 4. Candidate Diff预算

| 数据规模 | 首屏 | 完整结果 |
|---|---:|---:|
| 5000中文字符 | ≤500ms | ≤1.2s |
| 20000中文字符 | ≤1.2s | ≤3s |

要求：

- Diff可取消。
- 计算期间编辑器仍可响应。
- 优先完成结构匹配和可见区域，再补充完整字符Diff。
- 超过事件循环100ms的CPU任务使用Worker或分片。

## 5. SQLite预算

| 操作 | 目标 |
|---|---:|
| 打开常规项目 | ≤1s |
| 打开中型项目 | ≤3s |
| 单次Draft Patch事务 | P95≤150ms |
| Version创建（常规章节） | ≤300ms |
| Candidate采用 | 常规章节≤500ms |
| `quick_check`常规项目 | ≤2s |
| FTS普通查询 | P95≤200ms |
| FTS重建 | 显示真实进度，可取消或后台完成 |

中型项目Fixture：150万字、500章、150个实体、200条伏笔和合理数量Version。

## 6. Core并发预算

满足任一条件并持续可复现时，启动独立AI进程评审：

- AI运行期间编辑IPC P95>200ms。
- 自动保存P95>150ms。
- AI解析连续占用Core事件循环>100ms。
- 取消反馈>500ms。
- Diff或索引任务导致Renderer持续低于50fps。

触发评审不等于自动拆分；先确认根因和最小优化。

## 7. 流式事件预算

- Provider单Token输入必须合并为20—50ms批次或字符阈值批次。
- Renderer每秒处理事件数量必须有上限，慢消费时合并未发送delta。
- 切页后不继续渲染不可见正文，但任务状态保持更新。
- 长流式任务内存占用受Candidate最大输出和缓冲上限约束。

## 8. 显示与缩放预算

测试：

- 1280×800。
- 2560×1440：100%、125%、150%。
- 3440×1440和3840×1600。
- 混合DPI双显示器。

要求：

- SVG与文字无模糊缩放。
- 窗口跨屏后1秒内完成合理布局恢复。
- 21:9不把正文拉宽超过860 CSS px。
- 无整页水平滚动。
- 抽屉和弹层动画不超过200ms且不掉帧。

## 9. 备份与导入预算

- 日常备份在后台或空闲执行，不阻断编辑。
- 重大操作等待备份时必须显示真实阶段。
- 导入解析使用临时模型，预览可逐步显示。
- 导出使用流式/分块写入，避免复制整部作品多份到内存。
