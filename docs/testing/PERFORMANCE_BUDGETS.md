# WorldForge V1.0 性能预算

> 状态：Frozen  
> 目标：保证写作不中断、AI不拖慢编辑、目标显示环境稳定。

## 1. 测量原则

1. 使用P95而非单次最佳值。
2. 记录机器、系统、CPU、内存、存储、分辨率、缩放和数据规模。
3. 开发与正式构建分别记录，发布门以正式构建为准。
4. 模型网络耗时不作为应用性能指标；取消、流式渲染和本地处理计入。
5. 性能失败不能通过绕过安全与完整性检查修复。

## 2. 交互预算

| 指标 | 目标 | 阻断阈值 |
|---|---:|---:|
| 2K编辑器键入P95 | ≤50ms | >100ms |
| 普通按钮反馈 | ≤100ms | >250ms |
| 标签/抽屉切换 | ≤160ms | >300ms |
| 当前章查找首次结果 | ≤100ms | >300ms |
| 章节切换并显示正文 | ≤300ms | >800ms |
| 自动保存事务P95 | ≤150ms | >500ms |
| 编辑IPC往返P95 | ≤200ms | >500ms |
| AI取消确认 | ≤500ms | >1500ms |

## 3. 编辑器预算

- 常规章节：3000—8000字。
- 长章节：20000字。
- 极端Fixture：50000字，只保证可编辑，可关闭非必要实时统计。

目标：常规滚动≥50fps；输入、选择、复制和撤销无明显冻结；自动保存不阻塞输入；来源、锁定、弧光和检查标记不引发整章重渲染。

## 4. Candidate Diff

| 数据规模 | 首屏 | 完整结果 |
|---|---:|---:|
| 5000中文字符 | ≤500ms | ≤1.2s |
| 20000中文字符 | ≤1.2s | ≤3s |

Diff可取消；先完成结构和可见区域；连续阻塞超过100ms使用Worker或分片。

## 5. SQLite与FTS

| 操作 | 目标 |
|---|---:|
| 打开常规项目 | ≤1s |
| 打开中型项目 | ≤3s |
| Draft Patch事务 | P95≤150ms |
| Version创建 | ≤300ms |
| Candidate采用 | ≤500ms |
| `quick_check`常规项目 | ≤2s |
| FTS查询 | P95≤200ms |
| FTS重建 | 显示真实阶段，可取消或安全继续 |

中型Fixture：150万字、500章、150实体、200伏笔、人物弧光和合理数量Version。

## 6. Core并发门槛

满足任一条件且持续可复现时，才评审独立AI进程：

- AI期间编辑IPC P95>200ms。
- 自动保存P95>150ms。
- AI解析连续占用事件循环>100ms。
- 取消反馈>500ms。
- Diff或索引导致Renderer持续低于50fps。

## 7. 流式事件

- Provider增量合并为20—50ms或字符阈值批次。
- Renderer每秒事件数有上限，慢消费时合并未发送delta。
- 切页后停止不可见正文渲染，但任务状态继续。
- 长流内存受最大输出和缓冲上限约束。

## 8. 显示与主题

测试：1280×800；2560×1440 100/125/150%；3440×1440；3840×1600；混合DPI。

要求：

- SVG与文字清晰。
- 跨屏后1秒内合理恢复。
- 21:9正文不超过860 CSS px。
- 无整页水平滚动。
- 抽屉和弹层动画≤200ms。
- Theme A/B切换不触发正文重新加载或业务重复提交。
- 减少动态时印章表现不增加阻塞。

## 9. 备份与导入

- 日常备份在空闲执行，不阻断输入。
- 重大操作等待备份时显示真实阶段。
- 导入预览可渐进显示。
- 导出分块写入，避免多份完整正文同时驻留内存。
