# WorldForge 无障碍与可用性规范

> 状态：Frozen  
> 目标：保证键盘、焦点、对比度、中文输入、读屏和减少动效在核心创作流程中可用。

## 1. 基本要求

1. 所有核心功能均可通过键盘完成。
2. 颜色不作为唯一状态表达。
3. 焦点顺序与视觉顺序一致。
4. 弹层、抽屉和对话框正确管理焦点。
5. 中文输入法候选、全角标点和组合输入不被快捷键破坏。
6. 支持减少动效、高对比和界面缩放。
7. 读屏用户可以理解按钮、状态、错误和候选差异。
8. 无障碍实现不能绕过锁定、Revision和Candidate安全边界。

## 2. 键盘导航

### 全局

- Tab/Shift+Tab按视觉顺序移动。
- Esc关闭最上层可关闭弹层。
- Ctrl/Cmd+K打开命令面板。
- Alt+1—6可选切换一级入口，默认关闭时不与系统冲突。
- 快捷键支持自定义和恢复默认。

### 树与列表

- 上下键移动。
- 左右键展开/折叠树节点。
- Enter打开。
- Space选择。
- Home/End跳到首尾。
- 多选必须明确提示范围。

### 编辑器

- 保留系统和浏览器标准文本编辑快捷键。
- 自定义快捷键在输入法composition期间不触发。
- 锁定段落获得焦点时读出锁定状态。

## 3. 焦点管理

### 对话框

- 打开时焦点进入标题后的首个可操作控件。
- 焦点限制在对话框内。
- 关闭后返回触发按钮。
- 危险确认默认焦点放在取消或安全操作，不放在永久删除。

### 抽屉

- 打开后聚焦抽屉标题或首个控件。
- Esc关闭。
- 关闭后返回原控件。

### 页面跳转

- 跳到正文锚点时聚焦对应段落或问题标记。
- 从检查返回时恢复问题列表位置。
- 从候选审阅返回时恢复章节与滚动位置。

## 4. 可见焦点

- 所有交互控件必须有清晰焦点环。
- 焦点环与背景对比足够。
- 鼠标点击后可以弱化，但键盘导航时必须可见。
- 高对比主题加强边界和焦点，不依赖阴影。

## 5. 颜色与对比度

- 正文和主要UI文字满足长文本可读对比度。
- 次级文字仍需可读，不以极浅灰隐藏信息。
- AI来源、锁定、警告、危险、成功同时使用文字或图标。
- Diff新增、删除和修改提供符号或标签。
- 深色和护眼主题分别测试，不从浅色主题简单反相。

## 6. 语义与标签

每个控件需要：

- 可见标签或`aria-label`。
- 必要的`aria-describedby`帮助和错误关联。
- 正确角色和状态：expanded、selected、disabled、busy、invalid。
- 图标按钮提供动作名称，例如“锁定段落”“取消AI任务”。

禁止仅使用“更多”“点击这里”等缺乏上下文的读屏名称。

## 7. 状态播报

### 轻量播报

使用`aria-live="polite"`：

- 保存完成。
- AI阶段变化。
- 搜索结果数量。
- Candidate应用完成。
- 备份完成。

### 紧急播报

使用`aria-live="assertive"`：

- 保存失败。
- 项目转只读。
- 数据库损坏。
- 操作可能造成数据丢失。

避免频繁播报每个AI delta和每次字数变化。

## 8. 编辑器与读屏

- 章节标题和正文区域有明确语义。
- 每个段落保持标准文本编辑体验。
- 来源和锁定信息通过可选描述提供，不在每个字符间插入噪声。
- SceneBeat分隔有标题语义，但不干扰连续阅读。
- 校验问题可通过快捷键在问题间跳转。

## 9. Candidate Diff无障碍

提供三种等价方式：

1. 视觉Diff。
2. 结构化差异列表。
3. 单稿逐块对比。

每个差异项读出：

- 类型：新增、删除、修改、移动、冲突。
- 当前稿内容。
- 候选稿内容。
- 选择状态。
- 可用操作。

同步滚动不是唯一导航方式；提供上一差异和下一差异按钮。

## 10. 中文输入法

- composition期间不响应会提交、切页或改写正文的自定义快捷键。
- 输入法候选窗口不被浮动工具条覆盖。
- 全角引号、破折号、省略号和中文标点正常输入。
- 选区工具条等待composition结束后再出现。
- 自动保存只在组合输入完成后提交。

## 11. 点击目标

- 常规按钮最小可点击高度32px，主要按钮36—40px。
- 仅图标按钮至少32×32px。
- 高密度树列表可使用28—32px行高，但需支持界面缩放。
- 拖拽目标提供键盘替代操作。

## 12. 减少动效

当系统或应用启用减少动效：

- 抽屉、对话框取消大位移。
- 列表渐显改为直接显示或短淡入。
- 关闭所有持续动画。
- AI运行使用静态进度和阶段文字。
- 不影响任务状态和取消反馈。

## 13. 字体与缩放

- UI缩放90%—150%。
- 正文字号14—28px。
- 200%系统缩放下核心操作仍可完成。
- 不通过图片呈现重要文字。
- 用户选择本机字体失败时安全回退系统字体。

## 14. 表单

- 标签不只使用placeholder。
- 错误显示在字段附近并进入错误摘要。
- 提交失败后焦点移动到首个错误字段。
- 必填项明确，非必填项可以标“可选”。
- 滑块同时提供数值输入或键盘调整。

## 15. 拖拽替代

卷章、SceneBeat和列表排序除拖拽外，还要提供：

- 上移。
- 下移。
- 移到指定父节点。
- 输入或选择目标位置。

移动结果和影响预览可通过键盘完成。

## 16. 超时与任务

- 不设置会导致作者正文丢失的会话超时。
- AI和长任务无自动关闭对话框倒计时。
- Toast消失后，重要结果可在任务历史或状态区域找到。
- 自动关闭的轻Toast支持悬停或聚焦暂停。

## 17. 文案

- 使用直接、明确的动词。
- 禁用原因要说明具体条件。
- 危险操作说明影响范围和恢复能力。
- 新手文案使用“当前稿”“AI建议稿”“历史版本”。
- 专业术语可放在帮助、日志和专业模式详情中。

## 18. 测试矩阵

- 仅键盘完成新建项目、写作、AI生成、候选采用、定稿和导出。
- Windows Narrator、macOS VoiceOver至少完成核心路径抽查。
- 200%缩放与高对比主题。
- 减少动效。
- 中文拼音和五笔输入。
- 候选Diff的结构化差异列表。
- 错误、冲突、只读和恢复焦点流程。

## 19. 阻断标准

以下问题阻止UI任务标记Verified：

- 核心操作只能鼠标完成。
- 对话框焦点逃逸或关闭后焦点丢失。
- 锁定、危险和冲突只靠颜色表达。
- 中文输入法导致丢字、重复或误触快捷键。
- 保存失败无法被读屏或键盘用户发现。
- 200%缩放下核心按钮不可达。
