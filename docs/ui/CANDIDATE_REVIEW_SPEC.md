# WorldForge 候选审阅与冲突交互规格

> 状态：Frozen  
> 目标：让作者在长章节中高效比较、选择、融合和回退AI建议稿，同时保持Draft安全边界。

## 1. 核心原则

1. Candidate是AI建议稿，默认不进入当前稿。
2. 作者可以整稿采用、逐段采用、按SceneBeat采用、融合、丢弃或保留待后续处理。
3. 候选比较不以单一评分替代作者判断。
4. 冲突必须在提交前解决，Revision变化不能静默覆盖。
5. 应用Candidate是一个原子Draft事务，可整体撤销。
6. partial Candidate始终显示“不完整”，不能直接定稿。

## 2. 入口

- AI任务完成后的“查看建议稿”。
- 右侧AI标签的候选历史。
- 章节历史中的未处理Candidate。
- 快速改写升级为结构性改写。
- 融合任务完成后。

进入前先提交当前未保存Draft编辑。保存失败时阻止进入采用流程，但可以只读查看候选。

## 3. 页面结构

```text
顶部栏
├─ 章节标题 / 任务类型 / 候选状态
├─ 候选切换
├─ 视图切换
├─ 场景导航
└─ 退出 / 重新生成

主体
├─ 当前稿
├─ 候选稿
└─ 差异与选择控件

底部操作栏
├─ 已选择段落
├─ 冲突与锁定摘要
├─ 保存选择
├─ 应用到当前稿
└─ 丢弃/保留候选
```

## 4. 视图模式

### 双栏

适合宽屏和中等长度候选。左当前稿，右候选稿。

### 上下对照

适合1280×800和窄屏。上当前稿，下候选稿，可按块联动滚动。

### 单稿专注

一次显示当前稿或候选稿，页边显示差异与切换按钮。适合长时间通读。

### 只看差异

折叠未变化块，显示前后少量上下文。适合返修和局部候选。

### 候选并排

多个T0骨架可三列卡片比较；完整T1正文首版只支持两稿并排，更多候选通过切换查看，避免屏幕拥挤。

## 5. Diff呈现

优先级：

1. `logicalBlockId`对应。
2. 块新增、删除、移动、拆分、合并。
3. 块内中文字符差异。

视觉：

- 新增：AI来源色细边和轻底纹。
- 删除：中性删除标记，避免大面积红底。
- 修改：字符级强调。
- 移动：显示来源位置和目标位置。
- 锁定：保留锁定线与图标。

颜色必须配合图标、文字和结构，不只依赖红绿。

## 6. 同步滚动

- 默认按对应块同步。
- 对应关系断裂时以最近锚点同步。
- 用户可解除同步独立阅读。
- 场景导航跳转时两侧定位对应SceneBeat。
- 大段新增或删除不强行保持像素级同步。

## 7. 选择单位

支持：

- 整个Candidate。
- SceneBeat。
- 完整正文块。
- 冲突项。

V1不支持逐字符拼接采用，避免形成不可预测文本和复杂来源记录。作者可以进入手动合并编辑器处理细粒度内容。

## 8. 段落操作

每个候选块提供：

```text
采用此段
保留当前段
加入手动合并
查看生成依据
```

采用状态可批量切换：

- 本SceneBeat全部采用。
- 本SceneBeat全部保留当前。
- 采用全部非冲突段。
- 清空选择。

锁定块的采用控件禁用并说明原因。

## 9. 骨架比较

T0候选卡片显示：

- 倾向名称。
- SceneBeat顺序。
- 核心冲突升级。
- 信息释放。
- 结尾钩子。
- 必选节拍覆盖。
- 风险摘要。

操作：

- 选为T1基础。
- 编辑骨架后继续。
- 按节拍融合。
- 丢弃。

不显示“AI评分最高”作为默认选择。

## 10. 章节候选摘要

候选顶部摘要只显示可解释信息：

- 字数。
- 生成倾向。
- 基础Revision。
- 完整/部分状态。
- 必选节拍覆盖。
- 锁定块跳过数量。
- 校验风险数量。
- 模型与支持等级。

文风匹配分数等AI判断放入可展开详情。

## 11. 融合

### 节拍融合

左侧SceneBeat树为每个节拍选择来源Candidate。

### 骨架与血肉

允许设置：

```text
结构来源：Candidate A
表达来源：Candidate B
保留当前稿段落：指定块
```

融合先生成新的merge Candidate，仍需审阅，不直接写入Draft。

### 过渡修补

系统只对拼接缝隙生成必要过渡。过渡段显示AI来源和生成原因。

## 12. 冲突类型

| 类型 | 原因 | 处理 |
|---|---|---|
| Revision冲突 | 生成后Draft整体变化 | 重新计算对应关系 |
| Hash冲突 | 对应块已被作者修改 | 保留当前、采用候选、手动合并 |
| 缺失块 | 当前稿或候选块不存在 | 确认新增或删除 |
| 锁定冲突 | 目标块已锁定 | 只能保留当前或先退出解锁 |
| 结构冲突 | 拆分、合并、移动无法自动判断 | 进入结构冲突视图 |
| 项目/章节失效 | 对象已关闭、删除或切换项目 | 停止采用，保留候选 |

## 13. 冲突页面

### 默认双区

```text
当前稿 | 候选稿
下方：选择结果和手动编辑区
```

### 高级三栏

```text
当前稿 | 候选稿 | 合并结果
```

只在专业模式或用户主动打开时使用三栏，防止13寸屏幕拥挤。

## 14. 手动合并

- 以当前稿副本作为起点。
- 可插入候选句段。
- 编辑结果仍是待提交临时合并内容。
- 保存时再次校验Revision和锁定。
- 来源记录为mixed，并关联Candidate ID。

## 15. 应用前摘要

点击“应用到当前稿”后显示简要确认区：

- 将修改多少块。
- 新增/删除/移动数量。
- 是否存在锁定跳过。
- 当前Revision是否仍匹配。
- 是否创建采用前Checkpoint。

无冲突时不增加额外模态确认；直接在页面底部确认区完成，减少摩擦。

## 16. 应用后反馈

- 返回写作工作台并定位第一个修改块。
- 显示“已应用X个段落 · 撤销”。
- 来源模式可追踪Candidate。
- 自动保存状态基于Core事务结果。
- 候选状态更新为accepted，未采用部分仍保留在记录中。

## 16.1 主题相关表现（不改变底层契约）

本节13—16描述的应用流程、`candidate.apply`事件、冲突处理和Checkpoint机制在所有视觉主题下完全一致，遵循ADR-007。主题差异仅限于：

- Theme A（安静编辑部）：应用按钮为标准按钮交互，反馈条为纯文字提示。
- Theme B（水墨印章，见`UI_SYSTEM_THEME_B.md`）：应用按钮替换为印章落款动效，反馈条可叠加简短的水墨表现，但文案内容、撤销入口位置和可操作性与Theme A一致。

禁止为Theme B单独实现一套应用确认流程、单独的冲突判定逻辑或单独的撤销机制；如实现时发现某个交互点无法只靠Token和动画资源满足视觉隐喻，应先提出ADR-007修订，而不是绕开写分支代码。

## 17. 回退

### 即时回退

Ctrl/Cmd+Z或“撤销本次应用”执行inverse patch。

### 重启后回退

在历史操作或Version页选择采用前Checkpoint恢复为新Draft。

### 限制

如果应用后作者继续进行了大量编辑，整体撤销可能形成新冲突，应进入回退预览，不静默覆盖后续修改。

## 18. partial Candidate

标识：

```text
不完整建议稿：任务在接收过程中停止
```

允许：

- 阅读。
- 复制。
- 局部采用完整且可判断的块。
- 继续生成。
- 以此为基础创建新候选。

禁止：

- 一键定稿。
- 隐藏不完整标记。

## 19. 性能策略

- 首先渲染可见SceneBeat和结构Diff。
- 字符Diff分片或Worker计算。
- 折叠未改块减少DOM。
- 同步滚动按逻辑锚点，不持续逐像素计算。
- 5000字首屏≤500ms，完整≤1.2s。
- 20000字进入渐进计算并可取消。

## 20. 快捷键

| 操作 | 快捷键 |
|---|---|
| 切换左右/上下 | Alt+1 / Alt+2 |
| 单稿切换 | Alt+3 |
| 只看差异 | Alt+D |
| 下一差异 | F7 |
| 上一差异 | Shift+F7 |
| 采用当前块 | Alt+A |
| 保留当前稿 | Alt+K |
| 退出 | Esc |

快捷键需可自定义并避开系统冲突。

## 21. 验收重点

- 13寸、2K 125%和21:9布局。
- 5000字、20000字和大量未改段。
- 锁定、Revision、Hash和结构冲突。
- partial Candidate。
- 应用、即时撤销、重启后回退。
- 同步滚动与解除同步。
- 键盘全流程和读屏标签。

## 22. M2-03最小桌面验收面

M2-03只交付验证安全闭环所需的功能面，不提前替代M5-05完整候选工作台：

- 对话框内选择Fixture Candidate，显示当前稿、候选稿、基础Revision、结构Diff摘要和字符Diff块数。
- 计算期间显示“取消计算”；取消请求沿Preload/Main/Core严格链路终止分片或Worker任务，不产生项目库写入。
- 提供整稿、完整块和SceneBeat选择。partial Candidate自动禁用整稿采用。
- Apply冲突逐项显示ConflictSet类型与消息，冲突时明确“Draft未改变”。
- 成功采用后显示新Revision和“撤销本次应用”；重启后通过持久化ApplyRecord重新加载回退预览。
- Checkpoint损坏或应用后存在人工编辑时禁用无条件回退，显示冲突且保持当前Draft。

双栏/上下/单稿/只看差异切换、同步滚动、快捷键、完整响应式与无障碍视觉验收仍归M5-05；M2-03不得以这些延期项阻塞底层安全链路，也不得宣称已完成完整候选工作台。
