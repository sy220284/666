# WorldForge 编辑器交互规格

> 状态：Frozen  
> 适用：章节正文编辑、选区工具、锁定、保存、撤销、局部AI操作和场景联动。

## 1. 编辑器目标

- 中文长篇输入稳定。
- 自动保存不打断写作。
- AI操作始终可撤销、可追溯。
- 锁定块不被任何自动操作修改。
- 正文区保持简洁，复杂能力按需出现。

## 2. 文档结构

编辑器节点：

```text
chapterDocument
├─ heading
├─ paragraph
├─ dialogue
└─ separator
```

每个正文块映射：

```ts
{
  logicalBlockId: string;
  source: 'manual' | 'ai' | 'mixed' | 'imported';
  locked: boolean;
  contentHash: string;
}
```

节点属性仅用于交互映射；权威值由Core确认。

## 3. 输入与中文输入法

- composition开始后暂停自动拆分Patch。
- composition结束后形成单一编辑事务。
- 输入法候选窗口期间不移动选区、不触发自动格式化。
- Enter、Backspace和Delete在中文输入状态下遵循浏览器原生行为。
- 快速连续输入不逐字调用IPC。

验收覆盖拼音、五笔、手写输入、全角标点和中英文混输。

## 4. 自动保存

触发：

- 800ms无输入。
- 切换章节前。
- 进入候选比较前。
- 创建Version或定稿前。
- 应用关闭前。

流程：

```text
Tiptap事务
→ 合并为Block Patch
→ draft.applyPatch(baseRevision)
→ Core确认
→ 更新本地已保存Revision
```

保存失败时：

- 保留当前窗口内容。
- 停止显示“已保存”。
- 提供重试和复制当前文本。
- 不继续提交依赖新Revision的AI采用。

## 5. 撤销与重做

### 普通编辑

使用ProseMirror历史栈，事务边界与自动保存Patch保持对应。

### 持久化高风险操作

Candidate采用、批量替换、拆章和并章使用独立持久化回退记录。

规则：

- Candidate采用后首次Ctrl/Cmd+Z整体撤销本次采用。
- 普通输入与Candidate采用之间保持清楚事务边界。
- 应用重启后通过历史操作和Checkpoint恢复，不依赖内存撤销栈。

## 6. 选区工具条

触发：选择非空正文，且选区不跨不支持的结构。

常用操作：

```text
改写
精简
扩写
调整语气
检查
锁定
添加批注
```

规则：

- 工具条不遮挡选区主要文本。
- 选择跨锁定块时，AI操作禁用并解释原因。
- 跨多个自然段默认进入结构性改写。
- 单段符合条件时进入快速改写。
- Esc关闭工具条并保留选区。

## 7. 快速改写

适用条件：

- 选区位于一个自然段内。
- 不跨SceneBeat边界。
- 不包含锁定内容。
- 预期长度变化不超过原文约50%。

交互：

```text
选中文本
→ 选择改写类型或输入短指令
→ 原文旁内联显示AI建议
→ [应用] [换一个] [取消]
→ 应用后显示[撤销]
```

要求：

- 发起到应用不超过2次主要点击。
- 原文在应用前保持不变。
- 应用后来源标记为AI或mixed。
- 当前Revision变化时重新校验，必要时升级冲突流程。

## 8. 结构性改写

触发：

- 跨段落。
- 跨SceneBeat。
- 整章操作。
- 大幅改变长度或结构。
- 系统无法确定快速改写安全范围。

结果进入完整Candidate审阅，不在编辑器内直接替换。

## 9. 锁定

### 锁定单位

V1以完整正文块为锁定单位。局部文字锁定延后，避免复杂的范围映射和冲突。

### 操作

- 正文工具条提供“锁定当前块/解锁当前块”，并同步`aria-pressed`状态。
- 默认快捷键为Ctrl/Cmd+Shift+L，composition期间禁用。
- 锁定后显示低干扰页边细线、浅色底和“已锁定”文字，不只依赖颜色。
- 解锁需要明确操作，不要求二次确认。

### 保护

- 编辑器阻止直接输入、删除、合并和移动。
- Core再次检查所有Patch。
- AI、替换、拆并章和跨章移动均经过LockGuard。
- Core发现冲突时拒绝整批Patch，Renderer展示冲突块与跳过数量，不宣称部分保存。

## 10. 块拆分与合并

### Enter拆分

- 在段落中间Enter形成两个逻辑块。
- 原块保留logicalBlockId的一侧，另一侧获得新ID。
- 拆分规则由editor-core冻结并测试。

### Backspace合并

- 在块首Backspace可与前块合并。
- 任一块锁定时拒绝合并。
- 合并后的logicalBlockId继承规则稳定且可预测。

Candidate Diff必须理解拆分和合并关系。

## 11. 块移动

- 正文内拖动默认关闭或放在块菜单中，避免误操作。
- 开启时显示目标插入线。
- 锁定块不能移动。
- 跨SceneBeat或跨章节移动进入结构操作预览。
- 移动作为单一Patch事务，可撤销。

## 12. 粘贴

### 普通粘贴

保留：段落、换行、标题层级、基础粗体/斜体（若项目允许）。

移除：字体、字号、颜色、背景、脚本、网页布局、隐藏内容和不支持节点。

### 纯文本粘贴

提供Ctrl/Cmd+Shift+V。

### 大文本粘贴

超过阈值时提示：

- 直接粘贴到本章。
- 打开导入分章预览。

不自动将长文本拆成多章。

## 13. 查找与替换

- Ctrl/Cmd+F打开当前章查找。
- 命中高亮不改变正文数据。
- 替换全部前展示命中数和锁定跳过数。
- 跨章节操作跳转到检查工作台的全项目搜索。

## 14. 场景与正文联动

- SceneBeat可在正文页显示轻量分隔和目标摘要。
- 规划中移动SceneBeat不自动移动正文块。
- 作者可将选中段落关联到SceneBeat。
- 删除SceneBeat只移除规划关联，正文保留。
- 跨章移动已有正文关联时进入影响预览。

## 15. 来源模式

默认隐藏细节，只显示轻微来源标记。

开启来源模式后显示：

- 手动。
- AI。
- 混合。
- 导入。
- 来源Candidate或Version。

作者手动编辑AI段落后，来源更新为mixed；不进行逐字符作者占比伪精确统计。

## 16. 章节标题与元信息

- 标题在正文上方独立编辑。
- 标题保存与正文Patch可以同一事务或独立明确事务。
- 章节状态、目标字数和SceneBeat不嵌入正文文档。
- 正文导出时按格式渲染标题。

## 17. 快捷键

| 操作 | 默认 |
|---|---|
| 保存 | Ctrl/Cmd+S |
| 撤销/重做 | Ctrl/Cmd+Z / Shift+Ctrl/Cmd+Z |
| 查找 | Ctrl/Cmd+F |
| 沉浸写作 | Ctrl/Cmd+Shift+F |
| 快速改写 | Ctrl/Cmd+Enter（有选区时） |
| 锁定段落 | Ctrl/Cmd+Shift+L |
| 打开命令面板 | Ctrl/Cmd+K |

快捷键冲突需在设置中提示。

## 18. 长章节降级

超过性能阈值时可逐步关闭：

- 非可见区域实时来源渲染。
- 非必要语法高亮。
- 实时全章统计。
- 非当前段落校验装饰。

不得关闭：

- 自动保存。
- Revision和Hash。
- 锁定保护。
- 撤销。
- 正文可读与可编辑。

## 19. 失败与恢复

- Core断开：编辑器进入“本地未提交”状态，允许复制文本，不继续假装保存。
- Patch冲突：重新加载相关块并提供保留当前窗口文本的合并入口。
- 章节已删除：停止写入，提示恢复或另存为新章节。
- 项目只读：编辑器不可输入，允许复制和导出。

## 20. 测试重点

- 中文IME。
- 锁定块所有编辑路径。
- Enter拆分和Backspace合并。
- 快速改写应用与撤销。
- 自动保存失败与恢复。
- 长章节性能。
- 切章、重开和窗口恢复。
- 2K 125%与深色主题下选区工具定位。

## 21. M1-04实现映射

- Renderer使用Tiptap/ProseMirror呈现paragraph、dialogue、heading、separator四类顶层正文块；Tiptap JSON仅为窗口状态，关闭重开时始终从DraftBlock重建。
- Enter拆分保留左块logicalBlockId，右块使用临时clientBlockId并在保存时由Core生成logicalBlockId；块首Backspace合并后保留前块logicalBlockId。
- composition期间禁用Enter、Backspace、Delete等破坏性结构键和保存；composition结束后只标记未保存，不在每个输入增量上跨IPC提交。
- 网页粘贴移除脚本、样式、布局节点和任意属性，只保留段落、标题、对话与分隔线语义；纯文本换行映射为正文段落。
- 本地撤销/重做和每章选区位置保存在ProseMirror/Renderer会话中；保存成功后只同步Core返回的块ID元数据，不清空本地历史与选区。
- M1-04阶段仅交付手动保存和只读复制；后续任务已将保存入口统一为800ms自动保存和Block Patch。

## 22. M2-01实现映射

- 当前块可通过正文工具条或Ctrl/Cmd+Shift+L锁定与解锁；按钮公开pressed状态，锁定块同时具有边线、底色、文字和无障碍标签。
- Tiptap LockGuard过滤任何会修改、删除、移动、拆分或合并锁定块的事务；Core对Patch和内部快照入口再次执行同一安全约束。
- Core锁定冲突保持Patch原子性，返回冲突类型、logicalBlockId和整批跳过数量；关闭项目再打开后锁定属性从项目数据库重建。
