# WorldForge 交互状态规范

> 状态：Frozen  
> 目标：统一所有页面的反馈、错误、任务、冲突和恢复行为，避免不同模块各自定义状态提示。

## 1. 状态优先级

```text
P0 数据安全：保存失败、数据库损坏、恢复失败、跨项目拒绝
P1 用户主动任务：AI生成、导入、导出、备份、Diff、索引重建
P2 待决策：候选、冲突、状态提案、低风险校验、备份提醒
P3 信息：字数、连接状态、最近保存时间、模型支持等级
```

同一区域同一时刻只显示最高优先级状态。首页主动提示最多2条。

## 2. 状态展示位置

| 状态             | 位置                                       |
| ---------------- | ------------------------------------------ |
| 保存状态         | 写作工作台底部状态栏、必要时顶部安全Banner |
| AI和长任务       | 全局任务条、发起页面局部进度               |
| 字段错误         | 字段附近                                   |
| 页面加载失败     | 页面主体ErrorState                         |
| 冲突             | 独立冲突视图或页面内冲突区                 |
| 数据损坏/只读    | 持续Banner与恢复入口                       |
| 短时成功         | 轻量Toast或状态文字                        |
| 删除、覆盖、恢复 | 明确确认对话框                             |

## 3. 空状态

空状态必须回答三件事：

1. 当前为什么为空。
2. 作者下一步能做什么。
3. 是否可以跳过。

示例：

- 无章节：“创建第一章”与“导入已有作品”。
- 无人物：“先添加主角”与“暂时跳过”。
- 无Candidate：“生成AI建议稿”或继续自主写作。
- 无检查问题：“当前未发现需要处理的问题”，不使用庆祝动画。

禁止用空白页面、通用“暂无数据”或强制完成设定阻断写作。

## 4. 加载状态

### 短加载（预计小于300ms）

不显示转圈，保留原内容并禁用重复提交。

### 中加载（300ms—2s）

使用局部Skeleton或细进度条，避免整页遮罩。

### 长任务

显示：

- 真实阶段。
- 已运行时间。
- 可计算时的完成量。
- 取消入口。
- 任务是否可以切页继续。

## 5. 保存状态

状态枚举：

```text
未修改 | 有未保存修改 | 保存中 | 已保存 | 保存失败 | 只读
```

规则：

- “已保存”可在2—3秒后弱化为最近保存时间。
- 保存失败必须持续显示，直到成功重试或进入只读。
- 保存中不阻断输入。
- 关闭窗口时存在未提交修改，必须等待提交结果或提示风险。
- Renderer只在Core确认事务成功后显示已保存。

## 6. 成功反馈

适用：设置保存、备份完成、导出完成、Candidate采用、恢复副本完成。

规则：

- 高频操作使用状态文字或轻Toast。
- 需要下一步动作时显示结果页或操作按钮。
- 成功色不用于评价AI文本质量。
- Toast默认3—5秒，可手动关闭；重要结果不只存在Toast中。

## 7. 失败反馈

错误信息结构：

```text
发生了什么
哪些数据受到影响
当前数据是否安全
用户可以执行什么动作
诊断ID或查看详情
```

示例：

```text
本次保存未完成。当前编辑内容仍保留在窗口中，项目数据库没有写入半成品。
[重试保存] [复制当前文本] [查看详情]
```

禁止：

- 只显示“操作失败”。
- 向用户直接展示SQL、堆栈、密钥或完整本地路径。
- 对不可重试错误提供无效的“重试”。

## 8. 取消状态

用户主动取消不使用红色错误。

### AI取消

- 首先显示“正在停止”。
- 完成后显示是否存在可保存的部分结果。
- 已保存partial Candidate时明确标记“不完整”。

### 导入/导出/备份取消

- 明确当前阶段是否可取消。
- 取消后清理临时文件。
- 已经原子提交的事务不伪装为已撤销。

## 9. 冲突状态

冲突类型：

- Draft Revision变化。
- 块Hash变化。
- 块已删除或移动。
- 锁定状态变化。
- 替换计划过期。
- 导出目标已存在。
- 恢复目录冲突。

冲突必须提供上下文与决策，不能降级为Toast。

### 文本冲突操作

```text
保留当前稿
采用候选稿
手动合并
跳过此项
```

提交前再次校验Revision。

## 10. 只读状态

触发：

- 数据库完整性失败。
- Schema高于当前应用支持版本。
- Migration失败。
- 项目目录不可写。

界面行为：

- 持续显示只读Banner。
- 禁用所有写入和AI采用操作。
- 允许浏览、搜索和安全导出。
- 提供备份恢复、更新应用或另存副本入口。

## 11. 数据风险确认

需要强确认：

- 永久删除。
- 覆盖已有导出文件。
- 放弃未保存的临时AI输出。
- 清理非定稿Version。
- 删除备份。
- 执行数据库修复或迁移。

确认对话框必须说明：

- 对象名称与范围。
- 是否可恢复。
- 是否已创建恢复点。
- 主操作的具体动词。

禁止使用含义模糊的“确定”。

M2-04结构操作先显示移动块数、字符数、源/目标变化、锁定阻断和恢复语义，用户确认后才创建恢复点并提交。永久删除必须先显示影响数量和引用阻断，再要求输入完整对象标题；取消或标题不匹配时不创建恢复点、不写项目库。危险操作默认焦点仍在安全/取消路径。

## 12. AI任务状态

推荐阶段：

```text
排队中
整理本章信息
连接模型
接收内容
解析结果
保存AI建议稿
检查建议稿
完成
```

这些文案必须映射真实事件。没有可计算总量时显示阶段和时长，不显示百分比。

## 13. 连接状态

Provider连接状态只在AI页和设置页详细显示。顶部仅在当前AI任务需要连接时展示：

- 可用。
- 未配置。
- 认证失败。
- 服务不可达。
- 模型未验证。

离线编辑期间不持续展示“AI未连接”警告。

## 14. 校验状态

严重度：

- 阻断风险：数据或明确结构错误。
- 高风险：明显连续性冲突。
- 建议：文风、节奏或表达风险。
- 信息：统计与提示。

AI语义判断不得使用绝对“错误”文案，建议使用“可能存在冲突”“建议核对”。

## 15. 撤销反馈

- 普通编辑由标准撤销栈处理。
- Candidate采用后显示短时“已应用AI建议稿 · 撤销”。
- 批量替换和结构操作完成后显示恢复点信息。
- 重新打开应用后，短时Toast消失，但历史记录和恢复入口仍可用。

## 16. Toast、Banner、Dialog使用边界

| 组件            | 用途                                   |
| --------------- | -------------------------------------- |
| Toast           | 低风险、短时、操作已完成               |
| Banner          | 持续风险、只读、保存失败、迁移状态     |
| Dialog          | 明确决策、危险操作、少量表单           |
| Drawer          | 上下文工具和窄屏侧栏                   |
| Full-page state | 数据损坏、候选冲突、导入预览、恢复流程 |

## 17. 防重复提交

- 点击后立即进入pending状态。
- 同一requestId只执行一次。
- 按钮文案可改为“正在创建”“正在应用”。
- 不通过隐藏按钮掩盖任务状态。
- 任务允许并行时显示独立任务项；不允许并行时解释原因。

## 18. 恢复位置

页面返回或重试后恢复：

- 当前项目。
- 章节。
- 编辑器滚动和选区附近位置。
- 候选比较中的场景位置。
- 检查问题筛选和滚动位置。

数据源已变化导致位置无效时，跳转最近有效锚点并说明。

## 19. M3-09 Query/Command状态基线

- React业务面只通过统一Bridge Adapter访问Preload，不直接读取`window.worldforge`。
- Query以资源键和请求代次处理首次加载、替换刷新、取消、失败与成功；过期响应不得覆盖新选择。
- Command在同步与异步阶段都保持单次pending，拒绝重复提交；Core失败码原样进入页面状态，成功后再刷新权威Query。
- 只读状态集中传入规划、设定、结构、恢复和导入导出组件；浏览与安全导出保留，写入按钮统一禁用。
- 结构移动、永久删除、SceneBeat正文块移动、恢复和导入提交不得由刷新副作用触发，必须经过显式预览与确认链。
