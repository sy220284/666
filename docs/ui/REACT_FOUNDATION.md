# React UI基础

## 状态层级

| 层级 | 语义 | 示例 |
|---|---|---|
| P0 | 安全、恢复、只读与数据风险 | Core崩溃、恢复模式、保存阻断 |
| P1 | 阻断性任务与作者必须处理的动作 | 冲突裁决、必须确认的应用结果 |
| P2 | 当前页面或任务上下文 | 生成进度、当前选择、检查结果 |
| P3 | 短时反馈 | 已保存、已复制、已刷新 |

高优先级状态不得被低优先级反馈覆盖；同级状态优先展示持久项，再按时间与稳定ID排序。

## Design Token

M3-07新增React基础样式只使用集中CSS变量。主题只能改变表现值，禁止改变业务命令、可用动作或权威状态解释。

## Zustand边界

允许：一级路由、当前选择ID、侧栏模式、Dialog、任务显示、返回位置。

禁止：Project、Draft、Candidate、Version、Canon、EntityState及任何可作为提交基线的业务对象。

## 兼容阶段

M3-07 React Root保持最小且可观测，旧业务面继续工作。M3-08负责壳层、首页、项目与设置；M3-09负责规划、设定、结构与数据工具；M3-10负责写作、Version、Candidate及旧入口退役。
