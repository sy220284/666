# WorldForge 响应式布局、高分屏与曲面屏规格

> 状态：Frozen  
> 目标：保证1280×800最低可用，2560×1440完整体验，21:9曲面/超宽屏布局舒适，混合DPI切换稳定。

## 1. 基本原则

1. 所有布局以CSS像素和有效视口计算，不按物理分辨率硬编码。
2. 正文行宽独立于窗口宽度，最大860 CSS px。
3. 窗口越宽，增加上下文空间和留白，不拉长正文行宽。
4. 主要操作靠近正文工作区，避免在超宽屏远端放置高频或危险按钮。
5. 界面缩放、正文字号和正文宽度是三个独立设置。
6. 图标使用SVG，避免位图在125%与150%缩放下模糊。
7. 页面不得出现整页水平滚动；局部表格或Diff可在明确容器内处理。

## 2. 有效视口断点

| 视口宽度 | 模式 | 布局 |
|---|---|---|
| `<900px` | 紧凑 | 左右侧栏均为抽屉，正文单栏 |
| `900—1099px` | 窄屏 | 左侧栏可见/可收起，右侧栏抽屉 |
| `1100—1439px` | 标准 | 三栏可用，侧栏宽度较窄 |
| `1440—1919px` | 2K标准 | 完整三栏，正文760px |
| `1920—2559px` | 宽屏 | 三栏加宽，正文760/860px |
| `≥2560px` | 超宽 | 工作区限宽并支持居中/偏左/偏右 |

断点基于应用窗口的CSS像素，不等同于显示器物理像素。

## 3. 核心尺寸

| 区域 | 紧凑 | 标准 | 宽屏 |
|---|---:|---:|---:|
| 左侧栏 | 抽屉/220px | 240—280px | 280—320px |
| 右侧栏 | 抽屉/300px | 300—340px | 340—380px |
| 顶部栏 | 48px | 48—52px | 52px |
| 底部状态栏 | 28—32px | 30—34px | 32—36px |
| 正文版心 | 680px | 760px | 860px可选 |
| 页面外侧最小间距 | 12px | 16—24px | 24—40px |

侧栏支持拖动调整，拖动范围必须有限制，不能挤压正文到不可用宽度。

## 4. 1280×800最低可用

目标：完成新建、写作、AI快速操作、候选审阅、定稿和导出。

规则：

- 默认左侧目录240px，右侧栏进入抽屉。
- 顶部操作收敛到图标+命令菜单，保留保存和任务状态。
- 正文使用680px版心或占用剩余空间，最小可读宽度560px。
- 候选比较默认上下对照或单稿，不强制双栏。
- 对话框高度不超过视口，内容区可滚动，底部操作固定。
- 状态栏只显示保存、字数和当前任务，其他信息折叠。
- 任何核心按钮都不能被系统任务栏或窗口边界遮挡。

## 5. 2560×1440与缩放

### 100%

- 可使用完整三栏。
- 正文760或860px。
- 右侧栏340px。
- 候选比较双栏完整显示。

### 125%

有效CSS视口约为2048×1152，作为主要验收环境：

- 完整三栏。
- 正文760px。
- UI字号和点击目标不缩小。
- SVG、分隔线和1px边界按设备像素清晰处理。

### 150%

有效CSS视口约为1707×960：

- 保持三栏或根据用户侧栏宽度自动将右栏转抽屉。
- 候选双栏仍可用，空间不足时自动切上下。
- 对话框和菜单不越界。

## 6. 21:9曲面与超宽屏

目标：利用空间展示上下文，同时控制视线移动和颈部负担。

### 工作区定位

用户可选：

- 居中。
- 偏左。
- 偏右。

默认居中；单人坐姿偏向显示器一侧时可调整。设置只改变工作区整体位置，不改变正文数据。

### 最大工作区宽度

建议：

```text
工作区最大宽度 = 左侧栏 + 正文区 + 右侧栏 + 内部间距
约1600—1800 CSS px
```

多余空间作为安全留白，不无限拉开侧栏与正文。

### 操作距离

- 右侧栏紧邻正文。
- “应用候选”“永久删除”“恢复”等重要操作在内容区底部或近侧位置。
- 顶部全局设置可在窗口边缘，高频写作操作不可远离正文。
- Toast和任务条靠近工作区，不固定在物理屏幕最右角。

### 曲面视觉安全区

- 主正文和主要对话框位于中间约70%视觉区域。
- 边缘区域用于低频导航、留白和辅助状态。
- 不在强曲率边缘放置长文本、Diff细节和精确拖拽目标。

## 7. 混合DPI多显示器

保存：

```ts
{
  displayId: string;
  boundsDip: { x: number; y: number; width: number; height: number };
  scaleFactor: number;
  maximized: boolean;
}
```

恢复流程：

1. 查找原displayId。
2. 原显示器不存在时选择主显示器。
3. 使用DIP坐标恢复。
4. 校验窗口至少有主要区域位于可见工作区。
5. 根据新有效视口重新计算布局断点。

跨屏行为：

- Electron的缩放变化事件触发重新布局。
- 不手动对CSS尺寸重复乘scaleFactor。
- 拖动过程中避免频繁持久化，停止后保存。
- 菜单、Popover和选区工具条在新屏幕上重新定位。

## 8. 界面缩放

可选范围：90%—150%，推荐步进10%。

界面缩放影响：

- UI字体。
- 控件尺寸。
- 侧栏和面板。
- 图标。

界面缩放不影响：

- 正文字号设置。
- 导出字号。
- 项目正文内容。

缩放后重新判断断点，禁止通过CSS transform整体缩放导致文字模糊。

## 9. 正文字号与宽度

### 字号

14—28px，本地偏好保存。

### 宽度

- 窄：680px。
- 标准：760px。
- 宽：860px。
- 自适应：在上述范围内根据视口变化。

正文宽度和字号组合需限制每行中文字符数，避免大字号配窄版心造成过度换行，也避免小字号配宽版心造成长行。

## 10. 写作工作台响应规则

### 标准三栏

```text
左目录 | 中正文 | 右上下文
```

### 右抽屉

```text
左目录 | 中正文 | [右栏抽屉]
```

### 双抽屉

```text
[左目录抽屉] | 中正文 | [右栏抽屉]
```

抽屉打开时：

- 可覆盖正文部分区域。
- Esc关闭。
- 焦点限制在抽屉内。
- 关闭后恢复原焦点。

## 11. 候选比较响应规则

| 可用宽度 | 默认模式 |
|---|---|
| ≥1500px | 双栏 |
| 1000—1499px | 双栏紧凑或上下，按正文长度选择 |
| <1000px | 上下或单稿 |

用户手动选择视图后，本次会话保持，不在滚动中自动切换。

## 12. 表格、时间线和关系视图

- 表格优先列隐藏和详情抽屉，减少水平滚动。
- 时间线在窄屏使用列表。
- 关系图在窄屏提供实体列表和关系详情作为等价入口。
- 数据图视图不承担唯一操作路径。

## 13. 对话框与浮层

- 最大宽度使用`min(720px, viewport - 32px)`。
- 危险确认通常480—560px。
- 长表单改用独立页面或右抽屉。
- Popover检测边界自动翻转。
- 选区工具条不得越出可见区域或遮挡输入法候选框。

## 14. 性能

- Resize事件节流。
- 不在每个像素变化时重算大规模Diff或虚拟列表。
- 2K 125%下滚动≥50fps。
- 侧栏动画不触发正文整棵树重渲染。
- 主题和缩放切换允许短时重排，不能丢失选区和未保存文本。

## 15. 验收矩阵

必须截图与操作验证：

```text
1280×800 @100%
2560×1440 @100%
2560×1440 @125%
2560×1440 @150%
3440×1440 @100%
3840×1600 @100%或系统推荐缩放
双显示器：100% + 150%
有效视口1024×640
```

每种环境检查：首页、写作、右抽屉、候选比较、冲突、设置、导入预览、备份恢复和对话框。
