# WorldForge 页面规格

> 状态：Approved  
> 目标：定义每个核心页面的结构、主要操作、数据来源、状态和页面间关系，作为前端实现和E2E验收依据。

## 1. 应用启动页

### 目标

在无项目、最近项目存在、项目损坏和Core启动失败时提供明确入口。

### 结构

```text
顶部：应用名称 / 设置 / 帮助
主体：继续写作 / 新建项目 / 导入已有作品
下方：最近项目列表
底部：版本、数据位置、Core状态
```

### 主要操作

- 打开最近项目。
- 新建项目。
- 导入已有作品。
- 定位丢失项目路径。
- 打开设置。

### 数据来源

`app.sqlite`中的最近项目、应用设置和Provider元数据。

### 状态

- 首次启动空状态。
- Core启动中。
- Core失败。
- 最近项目路径丢失。
- 项目Schema过新。
- 项目完整性异常。

## 2. 新建项目向导

详细交互见`ONBOARDING_SPEC.md`。

### 步骤

1. 选择入口：快速开始、完整流程、导入已有、空白项目。
2. 基础信息：名称、频道、项目位置。
3. 故事核心：一句话、高概念、主角目标、核心冲突，可跳过。
4. 关键人物：最小人物卡，可跳过。
5. 第一章：目标、场景或直接进入编辑器。

### 完成结果

创建工作空间、项目数据库、第一卷、第一章和活动Draft。向导完成前不写入半成品项目，或以明确的可删除草稿项目处理。

## 3. 首页

### 目标

让作者快速继续当前作品，控制提醒数量。

### 新手布局

```text
继续写作主入口
项目进度：已完成的建书步骤
一条下一步建议
最近项目
新建/导入
```

### 专业布局

```text
最近项目列表
项目最近章节与修改时间
运行任务
数据风险或恢复提示
快速创建空白项目
```

### 规则

- 主动提醒最多2条。
- 不展示复杂图表和运营指标。
- 项目卡只显示名称、最近章节、字数、最后修改时间和健康状态。

## 4. 规划工作台

### 布局

```text
左：卷章与大纲树
中：当前节点编辑区
右：相关人物、设定、伏笔和目标摘要
```

### 中区内容

- 任务书表单。
- PlotNode详情。
- Chapter目标。
- SceneBeat列表或卡片板。
- 故事指南针。

### 主要操作

- 新建卷、章、剧情弧、SceneBeat。
- 拖动排序。
- 关联人物、地点和伏笔。
- 将正文片段转为SceneBeat。
- 从SceneBeat跳转到章节正文。

### 风险交互

- 移动已有关联正文的场景时展示影响预览。
- 删除规划节点不自动删除正文。
- 大纲变化后不自动调用AI重写。

## 5. 写作工作台

### 布局

```text
顶部：项目/章节标题、模式、全局搜索、任务、保存状态
左：卷章目录
中：章节标题、正文编辑器、底部状态栏
右：本章 / AI / 人物 / 设定 / 检查
```

### 正文区

- 纸面背景。
- 680/760/860px正文宽度。
- 段落页边来源和锁定标识。
- 当前选区浮动工具条。
- 章节标题和可选场景分隔。

### 底部状态

- 保存中 / 已保存 / 保存失败。
- 正文字符数、纯文字字数和目标进度。
- AI任务阶段。
- 当前检查风险摘要。

### 主要操作

- 输入、粘贴、查找、撤销重做。
- 锁定/解锁段落。
- 快速改写。
- 发起T0/T1或结构性改写。
- 打开候选审阅。
- 创建Version或定稿。
- 进入沉浸写作。

## 6. 沉浸写作视图

### 保留内容

- 返回按钮。
- 项目/章节标题。
- 正文。
- 保存状态。
- 字数。
- 可关闭的轻量AI入口。

### 隐藏内容

- 一级导航。
- 左右侧栏。
- 校验详情。
- 版本和历史列表。
- 非紧急状态提示。

### 行为

- Esc退出。
- 退出后恢复原工作台滚动和侧栏状态。
- P0保存失败仍可在沉浸视图显示。

## 7. 右侧栏页面

### 本章

- 章节目标。
- SceneBeat覆盖。
- 目标字数。
- 当前状态：写作、审阅、定稿。
- 尾快照和状态提案入口。

### AI

- 任务选择：骨架、扩写、续写、改写、融合、检查。
- 当前Provider、模型和支持等级。
- 约束摘要和高级参数。
- 运行中任务与历史候选。

### 人物

- 本章/当前场景人物。
- 当前状态摘要。
- 知情信息和声音样本。
- 跳转完整人物页。

### 设定

- 当前地点、物品、规则、势力和伏笔。
- 来源和当前有效状态。
- 跳转完整设定页。

### 检查

- 红/黄风险摘要。
- 问题列表和正文锚点。
- 运行本章检查。
- 忽略、降级、转待办。

## 8. 候选审阅页

详细规则见`CANDIDATE_REVIEW_SPEC.md`。

### 顶部

- 章节和任务类型。
- 候选切换。
- 视图：双栏、上下、单稿、只看差异。
- 退出和重新生成。

### 中部

- 当前稿与候选稿。
- 差异标记。
- 场景/节拍导航。
- 段落采用按钮。

### 底部

- 已选择段落数量。
- 冲突数量。
- 应用到当前稿。
- 保存为历史候选。
- 丢弃。

## 9. 冲突处理页

### 默认布局

```text
左：当前稿
右：候选稿
下方或侧边：作者选择结果摘要
```

高级布局可显示三栏：当前、候选、合并结果。

### 每个冲突项

- 冲突类型。
- 当前段落。
- 候选段落。
- 基线Hash变化说明。
- 选择当前、选择候选、手动合并。

### 提交

所有冲突解决后再次校验Revision并一次事务提交。

## 10. 历史版本页

### 内容

- Version时间线。
- 类型、标签、字数、来源Revision。
- 与当前稿比较。
- 恢复为新Draft。
- 导出该Version。

### 规则

- 不显示编辑Version按钮。
- 删除Version属于高级空间清理，定稿Version默认不可删除。

## 11. 设定工作台

### 左侧

实体类型和实体列表：人物、地点、势力、道具、能力、规则、事件、自定义。

### 中部

- 基础资料。
- Canon事实。
- 当前状态。
- 历史状态。
- 关系和引用章节。

### 右侧

- 正文证据。
- 相关伏笔。
- 知情信息。
- 冲突或状态提案。

### 规则

- Canon编辑必须显示作者操作来源。
- AI提案单独显示，不混入已确认字段。

## 12. 时间线与伏笔页

### 时间线

- 列表与简化时间轴切换。
- 事件起止、精度、人物、地点和依赖。
- 冲突筛选和正文来源。

### 伏笔

- 生命周期看板。
- 埋设、强化、部分揭示、揭示、取消。
- 回收窗口和超期提醒。
- 关联章节和依赖。

V1不展示复杂自动布局图作为主入口。

## 13. 检查工作台

### 左侧分类

- 本章问题。
- 全书问题。
- 连续性。
- 设定。
- 文风。
- 机械校对。
- 修订待办。

### 中部列表

每项显示严重度、标题、章节、锚点、简短依据和状态。

### 右侧详情

- 原文上下文。
- 预期依据。
- 建议。
- 跳转正文。
- 忽略、静音、降级、误报、转待办。

## 14. 搜索与替换页

### 简单模式

查询框、范围、结果列表和正文上下文。

### 高级模式

大小写、正则、实体类型、历史Version和词典选项。

### 批量替换

- 先生成ReplacePlan。
- 展示命中、锁定跳过和冲突。
- 创建恢复点后提交。

## 15. 导入预览页

### 步骤

1. 文件检查。
2. 编码或格式确认。
3. 分章预览。
4. 合并、拆分和重命名。
5. 导入摘要。
6. 提交。

预览期间项目数据库不改变。失败和取消后清理临时数据。

## 16. 导出页

- 选择范围：全书、卷、章。
- 选择Version：定稿、指定历史版本。
- 选择格式：TXT、Markdown、DOCX。
- 预览标题、顺序和字数。
- 选择目标文件。
- 导出进度和结果。

## 17. 备份与恢复页

### 三个区域

- 日常备份。
- 重大操作恢复点。
- 手动命名快照。

每项显示时间、类型、大小、验证状态和备注。

### 恢复流程

选择备份→选择新目录→验证→注册恢复副本→打开副本。

禁止提供默认“覆盖当前项目”主操作。

## 18. 设置页

### 通用

语言、启动行为、最近项目和默认模式。

### 编辑器

字体、字号、正文宽度、行高、段距、首行缩进、打字机模式。

### 外观与显示

主题、界面缩放、工作区位置、减少动效、高对比。

### AI连接

Provider列表、地址、模型、凭据引用、连接测试和支持等级。

### 备份

日常频率、保留数、空间上限和目标目录。

### 检查与词典

默认检查、静音规则和项目词典入口。

### 快捷键

查看和自定义；冲突检测与恢复默认。

### 高级

日志、诊断、数据库检查、FTS重建和开发信息。危险操作单独分组。

## 19. 页面通用状态

每个页面实现前必须列明：

- 空状态。
- 首次加载。
- 刷新加载。
- 部分数据加载失败。
- 完全失败。
- 只读。
- 无权限或边界拒绝。
- 任务运行中。
- 任务取消。
- 数据冲突。
- 恢复成功。

具体表现见`INTERACTION_STATES.md`。
