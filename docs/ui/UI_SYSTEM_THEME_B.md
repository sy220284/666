# WorldForge UI视觉系统 — Theme B：水墨印章

> 状态：Frozen（作者已确认色值、字体与动效方案）
> 视觉定位：水墨、宣纸、朱红印章
> 前提：必须遵守`UI_SYSTEM.md`§2的Token命名结构和`ADR-007`的主题/逻辑分离约束。本文件只定义Theme B专属取值和差异化组件表现，不重新定义交互契约。

## 1. 视觉目标

- 纸感：正文区背景模拟宣纸质感（暖白偏米，不用纯白），比Theme A更强调"书写介质"的存在感。
- 墨色：主文字取深灰暖调墨色，非纯黑，降低屏幕的冷硬感。
- 朱红克制使用：朱红只出现在"确认类动作"（候选采纳、章节定稿），不作日常UI强调色，避免和警示、危险色混淆，也避免视觉噪音。
- 字体：正文采用衬线字体表达书写质感，界面控件仍用无衬线字体，避免整屏衬线拖慢扫描速度。

## 2. Design Token取值

### 2.1 颜色

| Token | 取值 | 用途与约束 |
|---|---|---|
| `color.bg.canvas` | `#F3EEE2` | 应用画布底色，暖米白 |
| `color.bg.paper` | `#FAF7EE` | 正文纸面，比canvas更亮一档 |
| `color.bg.panel` | `#ECE6D6` | 侧栏、卡片背景 |
| `color.bg.elevated` | `#FFFFFF` | 弹窗、下拉等需要更高对比的浮层 |
| `color.text.primary` | `#2E2B26` | 墨色正文，非纯黑 |
| `color.text.secondary` | `#5B564C` | 次级文字 |
| `color.text.muted` | `#8C8577` | 占位符、辅助说明 |
| `color.border.subtle` | `#DDD4BE` | 常规分隔线 |
| `color.border.strong` | `#B8AC8E` | 需要强调的分隔/输入框边框 |
| `color.accent.primary` | `#5C6B57` | 日常UI强调色（链接、选中态、焦点环），取"苔痕"青灰绿，**不使用朱红**，避免和确认动作混淆 |
| `color.seal.primary` | `#A32B22` | 朱红，**仅用于**候选采纳/章节定稿的印章图形和相关小型图标，不用于正文、长文本或常规按钮底色 |
| `color.ai.source` | `#6E7A82` | AI来源标识，青灰色，与seal红明确区分，避免被误读为"已确认" |
| `color.warning` | `#B8860B` | 琥珀，警示 |
| `color.danger` | `#8B3A2E` | 赭红（比seal红更暗更棕），避免和确认动作的朱红混淆 |
| `color.success` | `#4F7860` | 低饱和松石绿 |

**对比度约束**：`color.seal.primary`和`color.danger`按设计不用作正文或长文本颜色，只用于图形、图标和大号数字/徽标，满足WCAG AA非文本对比度3:1即可；若实现中出现把seal红/danger赭红用作小号文本颜色的情况，须换用`color.text.primary`或各自的深色变体，不得直接复用本表数值——这一条专门用来避免此前UX评审中出现过的"warning/danger浅色模式对比度不达标"问题重演。深色模式和高对比模式的取值待M4-05实现阶段按同一原则单独推导，本文件不预先给出（避免和实际渲染环境脱节）。

### 2.2 字体

| Token | 取值 | 说明 |
|---|---|---|
| `font.serif.body` | Noto Serif SC | 仅用于正文编辑区和候选/版本阅读视图 |
| `font.sans.ui` | 沿用Theme A的界面无衬线字体 | 菜单、按钮、表单、侧栏等控件文字，不使用衬线 |
| 正文字号 | 与Theme A相同基准（不因换字体改变信息密度） | — |
| 正文行高 | 基准×1.15（例如Theme A若为1.6，则Theme B为1.84左右） | 衬线中文长时间阅读需要更多行间留白，具体倍数在实现阶段用真实字体度量校正 |
| 标题字重 | 500—600，不使用700以上粗体 | 避免衬线粗体在小尺寸下糊字 |

### 2.3 形状与阴影

- `radius.sm/md/lg`：比Theme A各收紧约2px，例如Theme A的`radius.md`若为8px，Theme B取6px，贴合传统版式的直线感，同时仍保留Token结构，不新增字段。
- `shadow.popover/dialog`：复用Theme A的层级和用途，只降低阴影的色相饱和度（阴影颜色带一点暖褐调，而不是纯黑半透明），避免冷灰阴影破坏纸感。

### 2.4 动效曲线

- `motion.fast/normal/slow`：时长与Theme A完全一致（不因视觉隐喻延长交互耗时，符合ADR-007）。
- 新增`motion.easing.seal`：`cubic-bezier(0.34, 1.56, 0.64, 1)`，带轻微过冲（overshoot），专用于印章下压动效，模拟物理按压回弹感；其余控件动效沿用Theme A的easing，不全局替换。

## 3. 候选采纳的印章动效

- **触发时机**：作者点击"应用到当前稿"确认后，`candidate.apply`命令成功返回时播放，与`CANDIDATE_REVIEW_SPEC.md`§16.1的表现层约束一致——动画只在成功回调后播放，失败或取消不触发。
- **视觉过程**（总时长180ms，落在`motion.normal`区间内）：
  1. 0–120ms：一枚朱红印章图形从按钮上方、带轻微初始旋转（约-6°）沿`motion.easing.seal`下压至目标位置，旋转回正。
  2. 120–180ms：印章落下瞬间触发一次极短的墨迹扩散（简单径向渐变缩放，不使用粒子系统，控制实现复杂度）。
  3. 180–480ms：被应用的段落背景泛起淡朱红高光（`color.seal.primary`的低透明度版本），300ms内线性回落到正常背景，起到"定位到修改处"的作用，替代Theme A纯文字提示条里的定位机制。
- **印章图形内容**：使用通用占位字样"稿"（简化篆书风格图形），不做成任何真实姓名或机构印章，避免过度定制化负担和潜在版权问题；后续如需支持作者自定义印文，作为V1.5可选项单独立项，不在V1.0范围内。
- **底层契约不变**：本节所有效果均为`candidate.apply`成功后的表现层反馈，命令本身、返回结构、冲突处理、Checkpoint机制与Theme A完全一致（ADR-007）。

### 无障碍与降级

- 系统开启"减少动态效果"或`prefers-reduced-motion`时，跳过下压和墨迹动画，直接进入180ms后的段落定位与文字提示，效果等同于Theme A的纯文字反馈，不影响功能可用性。
- 印章图形必须提供文字等效状态（如`aria-label="已应用候选"`），不得只靠图形传达"是否应用成功"。
- 动画渲染失败时，Patch应用结果不受影响，UI仍需正确显示采纳后的状态（与`CANDIDATE_REVIEW_SPEC.md`§16.1一致）。

## 4. V1.0范围确认

- Theme B在V1.0只提供浅色、深色两档对比模式；护眼、高对比模式延后至V1.5（与M4-05任务卡当前假设一致）。
- 印章动效只应用于"候选采纳"和"章节定稿"两个确认点，不扩展到其他常规按钮，保持朱红的克制使用原则。

## 5. 确认记录

- 色值、字号倍数、动效时长：作者已确认，按本文件执行。
- 印章占位字样：定为"稿"，简化篆书风格图形，真实视觉稿（图形绘制）在实现阶段单独出，本文件只定义约束和参数。
- V1.0不支持作者自定义印文，固定使用占位字样；自定义印文延后至V1.5评估。
