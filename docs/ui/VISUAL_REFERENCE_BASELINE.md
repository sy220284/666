## 2. 共同视觉原则

```text
安静编辑部
正文优先
低饱和中性色
AI来源蓝紫
少卡片、弱阴影
清晰边界、稳定状态
```

参考图冻结的是：

- 区域关系。
- 主要尺寸比例。
- 信息层级。
- 高频操作位置。
- 正文版心控制。
- AI、风险和状态的视觉语义。

参考图不冻结：

- 具体图标库。
- 最终中文字体渲染。
- 单个像素误差。
- 真实内容长度。
- 动画细节。

## 3. 应用壳基线

结构：

```text
左侧一级导航 72px
顶部项目与全局状态 52px
主工作区保持中性背景
项目卡只承载启动信息
```

要求：

- 首页只突出继续写作、新建和最近项目。
- 主动提醒最多2条。
- 不放复杂运营图表。
- 一级导航始终使用文字或Tooltip说明。

## 4. 写作工作台基线

结构：

```text
左侧卷章目录 260px
中部正文工作区自适应
正文纸面 760px
右侧上下文 320px
底部状态 32px
```

要求：

- 正文纸面居于视觉中心。
- 左右栏边界轻，不使用厚重卡片阴影。
- AI、人物、设定和检查在同一右栏内按标签切换。
- 保存失败等P0状态可覆盖普通状态。
- 21:9时整体工作区限宽，不把两侧推到物理屏幕边缘。

## 5. 候选审阅基线

结构：

```text
顶部：章节、候选、视图和退出
中部：当前稿与AI建议稿
底部：选择摘要、冲突、应用和保留
```

要求：

- 当前稿和候选稿地位平等，不用绿色暗示AI稿更优。
- 采用按钮靠近选择结果。
- 差异采用低饱和背景、边线和文字标识组合。
- 1280×800自动改用上下或单稿视图。
- 冲突必须进入专门交互，不以Toast代替。

## 7. 组件层级

### Level 0：正文

最高视觉优先级。背景、字号、行高和版心稳定。

### Level 1：当前任务

章节标题、保存状态、AI任务、候选应用。

### Level 2：上下文

卷章目录、人物状态、SceneBeat、检查摘要。

### Level 3：辅助

帮助、设置、历史、低风险统计。

不得让Level 3通过颜色、动画或尺寸压过正文。

## 8. 实现验收

- 首个AppShell、写作工作台和候选页PR必须附实现截图与对应SVG并排对比。
- 允许响应式差异，但必须说明差异原因。
- 视觉回归覆盖浅色、深色、护眼和高对比。
- 2K 125%和1280×800必须各保存一份截图。
- 页面不得为了像SVG而写固定绝对坐标；应使用CSS Grid/Flex和Token。

## 9. 后续扩展参考

规划工作台、设定工作台、检查工作台、导入预览和备份恢复的视觉基准，在相关页面首次实现任务中根据相同Design Token生成并冻结。不得在没有真实页面实现前批量制作大量无用原型。
